## 主动预警推送系统 (2026.07.12 建)
- **定位**：第二大脑"高价值消息主动预警"子系统，补齐行情/历史之外的"实时财经信息流"维度
- **双模式**：
  - 被动查询：用户问"高价值消息/高价值预警" → 我 `python core/alert_engine.py query 7` 读 `alerts/pending.json` 汇总返回
  - 主动推送：① automation `automation-1783838745907` 每2h系统级扫描落盘+播报 ② 对话活跃时我主动查 pending 播报新高价值预警
- **数据源**（WeStock MCP，只有我能调，纯Python守护进程调不了 → 拉取由对话层/automation驱动我）：
  - 主源 `data_hot` kind=news（全市场热新闻，含预告类/泛事件如火箭回收）
  - 确定性预告增强 `tool_event` names（earnings_release/lockup等，对应 data_events 42类id目录）
  - 扫描锚点 `data_calendar`（未来有事件的日期，不填date只返日期列表无内容）
- **价值评估**：impact(热度property归一) × urgency(时效) × relevance(关联) 加权，阈值 `alerts/rules.json`(默认55)，关键词甄别(预告类:将于/预告/即将/下周；重大类:突发/利好突袭/发射/回收/打击...)
- **落盘**：`alerts/pending.json`(待推送) + `alerts/history.json`(去重防重复轰炸)，复用"无真实反馈不标done"铁律
- **文件**：引擎 `core/alert_engine.py` | 规则 `alerts/rules.json` | 目录 `alerts/`
- **不碰**：`daytrade_system/*/news_monitor.py` `evolution/market/sector_monitor.py` `tools/finance_position_monitor.py` 等是别的AI建的监控脚本，本系统独立建，按多AI共处规则1只增不删

## 消息面信号分类铁律 (2026.07.15 回测定稿)
**核心原则**：个股该"看消息还是看图"，取决于其新闻因子的**衰减结构**，不取决于代码本身。用 fwd1/fwd2/fwd3 的秩相关 IC 判定（活跃日子集，无未来函数）：
- **新闻动量型** (fwd1 IC 显著>0, 衰减快)：事件日看消息面做**次日方向+波动辅助**。例: 昆仑300418 → mean_sentiment fwd1=0.27/fwd2=0.10/fwd3=0.03；article_impact fwd1=0.30。负面新闻→次日下跌最明确(负面偏好不对称)，正面不稳健拉升。
- **新闻滞后型** (fwd1≈0 但 fwd2/fwd3 显著>0)：新闻**不预判次日**，价格反应**滞后2-3日**。例: 蓝标300058 → mean_sentiment fwd1=0.03(≈随机)/fwd2=0.12/**fwd3=0.24**；article_impact fwd3=0.22；negative_ratio fwd3=**−0.29**(滞后负面更强)。⚠️ **关键修正(07-15二次回测)**: 滞后型即便秩相关显著, 也**须经"持有期策略P&L"验证才可称可交易**——蓝标按当日情绪方向持有1-5日, 全期限净值均≤0.74(各H累计−26%~−82%), **不可作为方向仓位触发**, 仅作定性上下文。
- **资金/技术驱动型** (全窗口 fwd1/2/3 IC 均不显著)：看图+四渠道资金，新闻仅同步印证，不预判。
- **回测口径**：因子取当日收盘可知新闻(date t)→次日收益 fwd1[t]=close[t+1]/close[t]−1；composite_index 在无新闻日恒为0须切活跃日子集；全样本多空净值会被上升趋势污染(假象)，须用活跃日趋势中性净值/五分位多空差。
- **实战定位**：新闻信号是**辅助**非单独依据(样本仅60-74活跃日、经济幅度中等)。昆仑事件日叠加情绪/影响级加权(持有1日策略净值1.85/t=2.14, 次日是全部收益来源)；蓝标事件日不调仓、滞后信号亦不可作方向触发(持有期策略全负), 仅定性参考。
- **泛化分类器**(脚本自动输出 decay_cls): 用 fwd1 vs fwd3/fwd5 的 |IC|≥0.10 判定 动量型/滞后型/持续型/资金技术型; 新增"持有期策略P&L"(H=1~5 趋势中性净值)作为**可交易性闸门**——IC显著≠可赚钱, 须过此关。
- **复算脚本**：`core/backtest_news_nextday.py`(读 data/news_db/staging/_raw/kline_*.json + data/news_index/exports/news_index_*_365.csv)；日K用 tdx_kline 串行拉(规避MCP限流)。
