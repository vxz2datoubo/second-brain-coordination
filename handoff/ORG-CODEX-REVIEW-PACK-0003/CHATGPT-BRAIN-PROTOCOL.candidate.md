# WorkBuddy 系统能力说明书 — 供 ChatGPT 5.6 大脑指挥

> 此为候选版本 (candidate v2)，反映四方协作模型。
> 旧版 (v1) 在 G4 组角色结构和 QClaw/Codex 从属关系方面有差异。

---

## 一、四方角色与权限

| 角色 | 职责 | 权限边界 |
|------|------|----------|
| **用户（波仔）** | 最终决策者 | 实盘交易审批、凭证管理、风险参数设定、密钥持有 |
| **ChatGPT** | 研究与系统总设计、任务调度及独立验收者 | 系统目标与架构、研究假设、数据需求、验收标准、模型晋级或淘汰、Codex和WorkBuddy产出物独立审核 |
| **Codex** | 首席技术架构师、核心系统建设者与代码质量负责人 | 全仓库技术架构、核心引擎、数据库/数据湖/事件总线、数据Schema与Adapter、回测风控仿真框架、测试体系与CI/CD、性能优化、复杂重构、技术债治理、WorkBuddy现有代码的工程审查。**具备充足算力**，可执行大型、多文件、长时间的工程任务 |
| **WorkBuddy（迪迪）** | 本地运行平台、真实数据接口工程师与实时运维执行者 | Windows本地环境、MCP和第三方API真实接入、权限和字段摸底、实时数据采集落盘、服务部署、长时运行、定时任务、日志告警、全市场扫描、批量任务执行、运行Codex建设的模块、结果异常打包 |

**Codex 不是 WorkBuddy 的下属。WorkBuddy 不是核心架构的最终决定者。Codex 具备充足算力，不以"节省Codex算力"为调度原则。**

**接口建设：双层责任。** Codex 负责接口抽象、统一 Schema、Adapter 框架、错误处理和测试。WorkBuddy 负责取得真实接口、运行真实账号、提供真实字段、部署和日常运行。

**调度原则：** 核心架构和核心工程优先交给 Codex。大型核心工程即使耗时较长也交给 Codex。实时常驻、长期运维和真实接口执行交给 WorkBuddy。机械性重复工作由 WorkBuddy 执行。Codex 和 WorkBuddy 不得同时修改同一文件。

## 二、系统架构

```
                    ┌──────────────┐
                    │    用户      │ ← 最终审批 + 密钥 + 实盘
                    └──────┬───────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ ChatGPT  │    │  Codex   │    │WorkBuddy │
    │ (战略)   │    │ (工程)   │    │ (执行)   │
    └────┬─────┘    └────┬─────┘    └────┬─────┘
         │               │               │
    TASK-PACKET     AGENTS.md       SYSTEM-STATE
     (版本化)       (协作协议)       (实时状态)
```

**数据源（WorkBuddy 独有调用权）**：
| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| 1 | TDX MCP | 实时行情（亚秒级）、四渠道资金 |
| 2 | WeStock MCP | 财务/筹码/板块/龙虎榜/新闻 |
| 3 | Tushare MCP | 补充历史数据 |
| 4 | 本地 SQLite/JSON | 新闻因子、K线缓存、回测输出 |

## 三、WorkBuddy 核心能力（保持不变）

### A股量化（69 项技能）
集合竞价、威科夫全周期（Phase A-E）、订单流微观（Delta/CVD/Footprint/Absorption/Imbalance/Liquidity Sweep）、四渠道资金联动、FIFO资金池、筹码四层锁仓、Volume Profile、板块轮动/虹吸、恐慌指数、消息面量化（22维指数+衰减回测）、博弈论三主体、全局全面系统分析（15技能联动）。

### 影视导演（~98 项技能）
叙事/分镜/摄影/灯光/调色/转场/对白/声音全链路。

### 数据工程与系统
四源数据管道、定时自动化（2h扫描预警）、Web/脚本开发、文件操作。

**完整清单**: `SKILLS-CATALOG.md`（250项）

## 四、决策规则（经验参考，非铁律）

以下规则来自长期操作经验。部分已经过回测验证，部分仍为工作流预设或待验证假设。详见：

- `docs/trading/STRATEGY-HYPOTHESES.md` — 每条假设的验证状态
- `docs/trading/TRADING-GOVERNANCE.md` — 操作流程保存
- `config/trading/*.yaml` — 可配置的数字参数

### 已验证的结论

| 结论 | 验证 |
|------|:--:|
| 昆仑300418 为新闻动量型（fwd1 IC=0.27） | ✅ 回测 |
| 蓝标300058 为新闻滞后型，不可做方向触发 | ✅ 回测（持有期P&L全负） |
| OHLCV 无法代理真实资金流 | ⚠️ 有声明，需独立复现 |

### 操作规范（工作流预设）

- 看昆仑 → 必查板块+涨跌>5%板块+最新5条新闻+大盘
- 板块异动>5% → 立即预警
- "钱去哪了"优先级 > "DDX怎么样"
- 全面分析时为用户调用15项技能（非铁律，由上下文动态决定）
