## 一、系统架构

```
daytrade_system/

│   ├── slot_manager.py       # 3槽位管理器（每日最多3笔未回笼）

│   ├── trade_logger.py       # 交易记录器（JSON持久化）
│   └── regime_clf.py         # 市场状态分类器（5种状态）
│
├── sentiment/                 # 情绪分析层
│   └── news_monitor.py       # 新闻情绪监测 + 板块联动分析
│
├── evolution/                 # 自进化层
│   ├── auto_tuner.py         # 自适应参数调优（Pearson相关分析）
│   ├── backtest_optimizer.py  # 回测优化器（跨日/阈值优化）
│   └── self_test.py          # 自我检验（每日验证系统正确性）
│
└── data/                     # 数据存储（自动创建）
    ├── trade_log.json         # 所有历史交易记录
    ├── slots.json             # 当前槽位状态
    ├── risk_state.json        # 风控状态
    └── param_evolution.json   # 参数进化历史
```

## 四、6因子信号模型

| 因子 | 权重 | 信号逻辑 |
|------|------|---------|
| F1_VWAP极端偏离 | 25% | 价格在VWAP 2σ上方→卖，1σ上方→+15分 |
| F2_RSI均值回归 | 20% | RSI>70→卖信号，RSI<30→买信号 |
| F3_量价分布 | 20% | 高量高价区卖出，低量低价区买回 |
| F4_动量衰竭 | 15% | 连续3根阴线→+25分衰竭信号 |
| F5_成交量Delta | 10% | 连续主动卖压→倒T卖信号 |
| F6_跳空缺口 | 10% | 高开>1%假突破→卖，低开>1%恐慌底→买 |

**置信度评级**：
- A++ (≥90分)：强烈建议，立即执行
- A (75-89)：建议执行
- B (60-74)：可执行，限额1笔/天
- C (45-59)：备选
- D (<45)：不执行

## 五、T+0 每日3笔策略

```
每日最多3笔未回笼卖出
├── 倒T：先卖后买（默认）
│   ├── 卖出后30分钟冷却期
│   ├── 目标接回：VWAP 或 S1 支撑
│   └── 止损：若涨超卖出价1%+则强制接回
│
└── 跨日仓位
    ├── 优先接回
    ├── 若开盘大跌>1%→立即接回（锁定利润）
    └── 14:50 强制平所有未完成槽
```

## 六、风控规则

| 规则 | 触发条件 | 后果 |
|------|---------|------|
| 硬止损 | 单笔亏损>2% | 无条件平仓 |
| 熔断 | 日累计亏损>2% | 全天停止T仓 |
| 连亏降仓 | 连亏2笔 | 降为1个槽 |
| 黑天鹅 | 跌破MA20 8% | 立即平所有T仓 |
| 冷却期 | 卖出后30分钟 | 不可再卖出 |

## 七、自进化机制

### 每周自动调参
1. 分析最近20笔交易中各因子分与盈亏的Pearson相关性
2. 相关性>0.3 → 该因子权重×1.05
3. 相关性<-0.2 → 该因子权重×0.85
4. 归一化后更新参数

### 每日自我检验
- 数据源连通性（TDX MCP ping）
- 槽位状态一致性（槽数=未回笼+已完成）
- 风控逻辑正确性（连亏/熔断/降仓）
- 参数合理性（权重和=1）

### 每周回测优化
- 测试持有1/2/3/5/10天的历史收益
- 找出最优持有天数
- 优化信号阈值
