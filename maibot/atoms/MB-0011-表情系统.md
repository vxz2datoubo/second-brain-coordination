# 表情系统

## 原子结论
表情系统让 MaiBot 能在聊天中使用、管理和学习表情包。用户文档关注功能行为，内部架构文档关注表情资源、触发逻辑和数据管理。

## 关键事实
- 表情能力属于聊天体验增强，与文本回复和多模态处理相关。
- 表情资源的归属和聊天流有关，不能粗暴写入错误的 session 维度。
- UI 中涉及聊天流展示时应优先展示实际名称，而不是裸 session_id。

## 研究入口
排查表情不出现、出现错场景、资源归属不对时，同时看表情系统内部架构、消息管线和聊天流解析。
