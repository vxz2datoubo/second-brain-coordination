# 插件组件类型速查

## 原子结论
插件能力不是单一入口，而是多种组件：Command 面向命令交互，Tool 面向模型/系统可调用工具，Hook 面向流程扩展点，Action 是 Legacy 组件，Event Handler 面向事件订阅，API/LLMProvider/MessageGateway 提供更深层集成。

## 组件边界
- Command：用户显式触发的命令式交互。
- Tool：可被推理或系统流程调用的能力。
- Hook：插入既有流程的扩展点。
- Event Handler：响应事件总线上的事件。
- LLMProvider：扩展模型提供能力。
- MessageGateway：扩展消息收发和平台接入能力。

## 使用建议
写插件前先明确触发来源：用户命令、模型工具调用、生命周期、事件订阅、平台消息，避免把所有逻辑塞进一个组件。
