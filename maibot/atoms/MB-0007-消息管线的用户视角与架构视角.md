# 消息管线的用户视角与架构视角

## 原子结论
消息管线描述一条外部消息从适配器进入 MaiBot、被规范化、匹配聊天流、进入推理/工具/记忆等处理阶段，并最终发送回复的过程。用户文档更关注“消息怎么被处理”，架构文档更关注模块边界和事件流。

## 关键事实
- 业务模块需要基于真实聊天流处理资源归属，不应随意自行计算 session id。
- 适配器把平台消息转换为内部消息结构。
- 后续处理会涉及回复决策、上下文、模型调用、工具、记忆和输出。

## 研究入口
排查“不回复”“记忆归属错”“聊天流展示不对”时，先沿消息管线追踪真实聊天流、平台、目标 ID 和消息类型。
