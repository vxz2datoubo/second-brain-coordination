# 故障排查的高频路径

## 原子结论
MaiBot 新手高频故障集中在网络、配置、API Key、端口、WebUI、MCP、机器人不回复、插件加载和数据库。排查应让具体错误暴露，而不是盲目加 fallback。

## 排查顺序
- 网络连通性：API、Git、插件安装、MCP 都可能受影响。
- 配置存在性和 TOML 语法。
- API Key、余额、模型名和 base_url。
- 端口占用，尤其 WebUI 和 WebSocket。
- 适配器是否收到平台消息，MaiBot 是否收到适配器消息。
- 插件依赖、Manifest 和兼容版本。

## 项目调试观
不要用兜底掩盖错误；应该精准定位核心问题，让异常及时、完整地暴露。
