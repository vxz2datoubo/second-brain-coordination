# 07 — 建议接入优先级

## 核心结论

| 问题 | 答案 |
|------|------|
| 哪个做实时主源？ | **TDX MCP** — 5档盘口 + 内外盘 + 涨停价; 快照字段最全 |
| 哪个做交叉校验/降级？ | **WeStock MCP** — 独有四渠道资金流; 作为资金维度和数据交叉验证源 |
| 本地文件比 MCP 更细？ | **无法确定** — TDX 客户端已关闭, 本地文件审计 blocked |
| 是否真正具备逐笔成交？ | **❌ 否** — 两个 MCP 都不提供; 需 TdxQuant/eltdx |
| 是否具备逐笔委托/撤单？ | **❌ 否** |
| 是否有集合竞价连续数据？ | **❌ 否** — 仅快照, 无线竞价流 |
| 当前能支持哪些指标？ | 价格/量能/盘口5档/内外盘/四渠道资金/涨停价/PE/PB/换手/量比/振幅/委比/阶段涨幅 |
| 不能支持哪些指标？ | 逐笔 Delta/CVD/OFI/Absorption/撤单率/Queue Imbalance/集合竞价未匹配量轨迹 |
| 接入统一 Adapter 前缺什么？ | ① exchange_time ② sequence_no ③ L2 逐笔流 ④ 集合竞价流 ⑤ 实时推送/订阅机制 |

## 建议的接入优先级

### P0 — 立即接入（已有能力）

| # | 组件 | 依赖 | 说明 |
|---|------|------|------|
| 1 | TDX MCP 快照 | 无 | 实时价格 + 5档盘口 + 涨停价 |
| 2 | WeStock 四渠道资金 | 无 | 特大/大/中/小单净流向 |
| 3 | WeStock 板块行情 | 无 | 板块排行和热度 |

### P1 — 短期补齐

| # | 组件 | 依赖 | 说明 |
|---|------|------|------|
| 4 | TDX 客户端恢复运行 | 用户操作 | 解锁本地文件审计 |
| 5 | TdxQuant / eltdx PoC | L2 账号权限 | L2 逐笔成交验证 |
| 6 | 分钟 K 线统一接口 | TDX + WeStock | 补齐 1/5/15/30/60 min 数据 |
| 7 | 统一时间戳层 | Codex 架构 | 注入 receive_time + local_seq_no |

### P2 — 中期建设

| # | 组件 | 依赖 | 说明 |
|---|------|------|------|
| 8 | 集合竞价 Adapter | L2 数据源 | 虚拟价/匹配量/未匹配量轨迹 |
| 9 | 逐笔成交 Adapter | L2 数据源 | Delta/CVD/OFI/Absorption |
| 10 | 订单簿重建 | L2 逐笔委托 | order_id + sequence_no |
| 11 | 跨源校验引擎 | P0 组件就绪 | 价格/量/时间一致性自动监控 |
