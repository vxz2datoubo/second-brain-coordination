# 注意力信号引擎实施任务清单

> 状态：Draft v0.1  
> 依赖蓝图：`docs/trading/ATTENTION-SIGNAL-ENGINE.md`  
> 默认交易状态：`NO_TRADE`  
> 首个试点主题：`tech.world_model`  
> 最后更新：2026-07-18

## 1. 使用方式

本清单用于 Codex、WorkBuddy、GPT 和用户在同一 GitHub 仓库内协作。任何任务开始前都要完成：

1. 能力检查。
2. 权限与合规检查。
3. 输入数据和验收条件确认。
4. 风险等级确认。
5. 任务完成后的测试、工件登记和交接。

没有证据、测试或验收记录的任务，不视为完成。

## 2. 角色分工

| 角色 | 主责 |
|---|---|
| 用户 | 账号授权、付费来源、资金和高风险操作、最终业务批准 |
| ChatGPT | 研究设计、来源核验、任务拆解、独立验收、跨模块协调 |
| Codex | 核心架构、契约、数据模型、管线、特征、测试、重构和文档维护 |
| WorkBuddy | 本地环境、真实平台接入、人工采集流程、调度、监控、故障和运行报告 |

## 3. 优先级定义

- `P0`：不完成就不能安全开始。
- `P1`：世界模型 MVP 必需。
- `P2`：自动化和回测增强。
- `P3`：规模化扩展。

## 5. Phase 0：治理与契约

### ASE-0001 建立模块治理文件

- 优先级：`P0`
- 主责：Codex
- 协作：ChatGPT
- 输出：
  - 模块级 `AGENTS.md` 或纳入现有交易治理文件
  - `NO_TRADE` 约束
  - 数据源合规原则
  - 工件登记流程
- 验收：
  - 明确 ASE 不进入订单关键路径
  - 明确谁能修改词包、数据源和公司映射
  - 明确真实数据不直接提交 Git

### ASE-0002 建立 Capability Registry 条目

- 优先级：`P0`
- 主责：WorkBuddy
- 协作：Codex
- 输出：每个候选平台的能力状态、账号、权限、入口、历史范围、导出方式和失败原因
- 首批平台：百度指数、微信指数、抖音指数、小红书、微指数、B站、知乎、GitHub、模型平台、论文源、研报源
- 验收：每个平台均有明确状态，不允许空白被理解为“可用”

### ASE-0003 建立 Unknown Registry 条目

- 优先级：`P0`
- 主责：ChatGPT
- 协作：WorkBuddy
- 输出：尚未验证的历史长度、接口、权限、口径、合规和成本问题
- 验收：未知项有负责人、下一步和截止条件，不静默消失

### ASE-0004 建立 ADR

- 优先级：`P0`
- 主责：Codex
- 输出：
  - 主动搜索与内容热度分层
  - 研究侧车边界
  - 世界模型单主题 MVP
  - 原始事实不可覆盖
- 验收：ADR 可被后续 PR 引用

## 6. Phase 1：世界模型人工 MVP

### ASE-0101 创建主题注册表

- 优先级：`P1`
- 主责：Codex
- 协作：ChatGPT
- 输出：`configs/attention/topics/tech.world_model.yaml`
- 字段：主题 ID、规范名、别名、相关词、排除词、子主题、版本、审核状态
- 验收：
  - 词包可版本化
  - 核心概念词与公司名、产品名分开
  - 修改词包不会覆盖历史版本

### ASE-0102 创建数据源注册表

- 优先级：`P1`
- 主责：Codex
- 协作：WorkBuddy
- 输出：`configs/attention/sources/*.yaml`
- 验收：每个指标包含官方名称、定义、单位、粒度、地域、能力状态和采集方式

### ASE-0103 设计人工采集模板

- 优先级：`P1`
- 主责：Codex
- 协作：WorkBuddy
- 输出：
  - CSV 或表单模板
  - 证据附件命名规则
  - 人工录入校验脚本
- 验收：
  - 缺失、零值、未收录、访问失败可区分
  - 每条数据有采集时间、采集人和来源工件

### ASE-0104 百度指数试采

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：世界模型词包的搜索指数、资讯指数、媒体指数和需求图谱可用性报告
- 验收：
  - 指标口径不混淆
  - 记录未收录关键词
  - 附官方页面证据

### ASE-0105 微信指数试采

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：入口、可查询词、历史范围、导出方式、账号限制和样例数据
- 验收：无法自动化也要形成稳定人工流程

### ASE-0106 抖音指数试采

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：升级后产品入口、搜索分、内容分、传播分、关联词和下载能力报告
- 验收：不沿用旧巨量算数假设，全部以当前实际界面验证

### ASE-0107 小红书试采

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：普通账号与商业账号可见能力差异、搜索指数和笔记供给样例
- 验收：明确哪些指标需要商业权限或授权 API

### ASE-0108 微博、B站、知乎试采

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：微指数、B站 Z 指数或站内公开数据、知乎热榜和主题数据的可用性报告
- 验收：热榜、内容指数和纯搜索指标分别记录

### ASE-0109 专业源试采

- 优先级：`P1`
- 主责：Codex
- 输出：GitHub、模型平台、论文源的世界模型主题样例
- 验收：
  - 仓库新增与单仓库 Star 增量分开
  - 下载量和累计收藏分开
  - 论文新增按日期和机构保存

### ASE-0110 生成首份人工周报

- 优先级：`P1`
- 主责：ChatGPT
- 协作：WorkBuddy
- 输出：世界模型注意力阶段周报
- 必含：
  - 主动搜索曲线
  - 内容供给曲线
  - 专业确认曲线
  - 数据质量和缺失
  - 初步扩散阶段
  - 反证和不确定性
  - `NO_TRADE`
- 验收：所有结论可追溯到原始数据和证据

## 7. Phase 2：半自动采集与数据质量

### ASE-0201 定义 JSON Schema

- 优先级：`P1`
- 主责：Codex
- 输出：Topic、Source、RawObservation、Feature、StageAssessment、MarketMapping、DataQualityReport 契约
- 验收：提供正例、反例和版本迁移测试

### ASE-0202 建立原始事实写入器

- 优先级：`P1`
- 主责：Codex
- 输出：只追加写入、幂等键、重复检测和修订机制
- 验收：相同采集任务重跑不会产生不可控重复

### ASE-0203 建立证据工件登记

- 优先级：`P1`
- 主责：Codex
- 协作：WorkBuddy
- 输出：截图、导出文件和页面快照的哈希、路径、来源和时间登记
- 验收：任意人工数据可回到对应证据

### ASE-0204 建立数据质量规则

- 优先级：`P1`
- 主责：Codex
- 输出：
  - 缺失率
  - 延迟
  - 异常跳变
  - 口径漂移
  - 单位变化
  - 权限状态变化
  - 证据缺失
- 验收：质量不合格数据不能进入线上评分

### ASE-0205 建立采集器基类

- 优先级：`P2`
- 主责：Codex
- 输出：统一适配器接口、重试、限流、日志和错误分类
- 验收：人工、官方 API、官方导出和公开页面四类采集方式可统一登记

### ASE-0206 配置调度与运行监控

- 优先级：`P2`
- 主责：WorkBuddy
- 输出：日级或更低频调度、失败告警、运行日志和日报
- 验收：任务失败不会静默，且不会自动绕过验证码或权限

## 8. Phase 3：特征、阶段和看板

### ASE-0301 单平台特征

- 优先级：`P2`
- 主责：Codex
- 输出：水平、变化、加速度、持续性、异常、反转、集中度特征
- 验收：每个特征有公式、输入、窗口和单元测试

### ASE-0302 跨平台特征

- 优先级：`P2`
- 主责：Codex
- 输出：搜索宽度、内容宽度、一致性、平台顺序和领先天数
- 验收：缺失平台不会被当成零值

### ASE-0303 扩散阶段基线模型

- 优先级：`P2`
- 主责：Codex
- 协作：ChatGPT
- 输出：S0-S6 的透明规则基线
- 验收：输出阶段、置信度、证据、反证和质量惩罚

### ASE-0304 注意力看板

- 优先级：`P2`
- 主责：WorkBuddy
- 协作：Codex
- 输出：分层曲线、平台矩阵、扩散路径、数据质量和事件标注
- 验收：不能只展示单一综合分

### ASE-0305 报告生成器

- 优先级：`P2`
- 主责：Codex
- 输出：日报、周报和专题报告模板
- 验收：报告自动附来源、时间、词包版本、质量状态和 `NO_TRADE`

## 9. Phase 1-3：A股映射

### ASE-0701 定义公司映射契约

- 优先级：`P1`
- 主责：Codex
- 输出：主题、技术环节、产业链位置、公司、证券、证据、相关性等级、有效期
- 验收：核心收入、研发储备、供应链、叙事和蹭热点可以区分

### ASE-0702 世界模型公司候选池

- 优先级：`P1`
- 主责：ChatGPT
- 协作：用户
- 输出：候选公司和证据清单
- 验收：不因股价上涨反向修改历史相关性

- 优先级：`P1`
- 主责：用户
- 协作：ChatGPT
- 输出：批准、拒绝、降级、过期流程
- 验收：未经审核的公司映射不得进入正式回测

### ASE-0704 市场特征接入

- 优先级：`P2`
- 主责：Codex
- 输出：收益、成交、换手、波动、扩散宽度、公告和研报特征
- 验收：复权、停牌、涨跌停和交易日对齐正确

## 10. Phase 4：回测和研究验证

### ASE-0801 建立主题样本集

- 优先级：`P2`
- 主责：ChatGPT
- 输出：世界模型之外的正例、反例和失败主题
- 验收：包含“热度高但无市场映射”和“市场先涨后有内容”的反例

### ASE-0802 领先滞后分析

- 优先级：`P2`
- 主责：Codex
- 输出：交叉相关、滚动领先天数和平台顺序统计
- 验收：报告不同窗口和市场状态下的稳定性

### ASE-0803 事件研究

- 优先级：`P2`
- 主责：Codex
- 输出：搜索突破、内容突破、专业确认和研报确认事件窗口
- 验收：包含基准、行业调整和多重检验说明

### ASE-0804 反作弊和混杂分析

- 优先级：`P2`
- 主责：Codex
- 协作：ChatGPT
- 输出：营销投放、重大新闻、平台改版、头部账号带动和重复内容控制
- 验收：异常主题有明确降权或排除理由

### ASE-0805 研究结论验收

- 优先级：`P1`
- 主责：ChatGPT
- 输出：可用、条件可用、不可用三类结论
- 验收：允许得出“该信号目前没有交易价值”的结论

## 12. 建议的第一个 Sprint

### Sprint 目标

在不写复杂爬虫、不接真实交易的前提下，把“世界模型”从概念讨论变成第一份可追溯数据资产。

### Sprint 范围

1. `ASE-0001` 至 `ASE-0004`
2. `ASE-0101` 至 `ASE-0103`
3. `ASE-0104` 至 `ASE-0108` 的能力试采
4. `ASE-0201` 的最小契约草案

### Sprint 完成定义

- 世界模型主题词包已经版本化。
- 至少 5 个平台完成真实可用性验证。
- 至少 3 个主动搜索或平台指数源有证据样例。
- 人工采集模板可以被第二个人复核。
- 未知和不可用项被明确登记。
- 没有任何虚构数据。
- 没有任何真实交易动作。
