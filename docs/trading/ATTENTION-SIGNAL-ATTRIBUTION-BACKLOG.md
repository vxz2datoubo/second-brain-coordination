# 注意力异常归因与歧义消解实施清单

> 状态：Draft v0.1  
> 依赖蓝图：`docs/trading/ATTENTION-SIGNAL-ATTRIBUTION.md`  
> 父模块：注意力信号引擎（ASE）  
> 默认交易状态：`NO_TRADE`  
> 首个案例：微信指数 `Matrix` 异常归因  
> 最后更新：2026-07-18

## 1. Epic 总览

| Epic | 名称 | 目标 | 优先级 |
|---|---|---|---|
| ASA-E00 | 治理与数据契约 | 固化归因状态、证据等级和禁止强行归因规则 | P0 |
| ASA-E01 | 关键词歧义注册 | 建立关键词、语义实体、主题三层映射 | P0-P1 |
| ASA-E02 | 异常检测 | 识别单日尖峰、持续抬升和数据伪异常 | P1 |
| ASA-E03 | 候选事件采集 | 收集异常窗口内可能的驱动事件 | P1 |
| ASA-E04 | 实体消歧与内容聚类 | 区分目标主题和同名无关内容 | P1-P2 |
| ASA-E05 | 证据评分与归因 | 候选原因排序、反证检查和置信度输出 | P1-P2 |

## 2. Phase 0：治理与契约

### ASA-0001 建立归因状态枚举

- 优先级：`P0`
- 主责：Codex
- 输出：
  - `CONFIRMED_SINGLE_CAUSE`
  - `CONFIRMED_MULTI_CAUSE`
  - `LIKELY`
  - `AMBIGUOUS`
  - `INSUFFICIENT_EVIDENCE`
  - `UNRELATED_SPIKE`
  - `DATA_ARTIFACT`
- 验收：所有异常记录必须有状态；状态不得为空。

### ASA-0002 建立证据等级

- 优先级：`P0`
- 主责：Codex
- 协作：ChatGPT
- 输出：官方公告、官方账号、可信媒体、平台原始内容、转载和弱来源的等级规则。
- 验收：低等级证据不能单独支撑高置信度结论。

### ASA-0003 建立禁止强行归因规则

- 优先级：`P0`
- 主责：Codex
- 输出：归因门禁和 `INSUFFICIENT_EVIDENCE` 回退路径。
- 验收：候选原因不足时系统仍能正常生成报告，不得自动补故事。

### ASA-0004 扩展 ADR

- 优先级：`P0`
- 主责：Codex
- 输出：
  - 高歧义词不得直接进入主题分数
  - 所有异常必须附归因状态
  - 允许原因未确定
  - 归因不进入订单关键路径
- 验收：后续实现 PR 引用对应 ADR。

## 3. Phase 1：关键词歧义注册

### ASA-0101 定义关键词契约

- 优先级：`P1`
- 主责：Codex
- 输出：`KeywordDefinition`、`KeywordSense` JSON Schema。
- 字段至少包含：
  - 原始字符串
  - 规范写法
  - 语言
  - 关键词角色
  - 歧义等级
  - 候选实体
  - 上下文词
  - 排除词
  - 生效版本
- 验收：支持一词多义和一实体多词。

### ASA-0102 创建 Matrix 关键词配置

- 优先级：`P1`
- 主责：Codex
- 协作：ChatGPT
- 输出：`configs/attention/keywords/tech.world_model.matrix.yaml`
- 必含：
  - `Matrix`
  - `Matrix 3.5`
  - `Matrix 3.5 昆仑万维`
  - `世界模型`
  - `World Model`
  - `昆仑万维`
  - `Riemann`
  - `黎曼模型`
  - 电影、数学、机器人等竞争语义
- 验收：`Matrix` 标记为 `HIGH`，锚点组合词标记为 `LOW`。

### ASA-0103 建立实体字典

- 优先级：`P1`
- 主责：Codex
- 输出：公司、产品、模型、电影、数学概念、机器人公司和其他同名实体的初始字典。
- 验收：每个实体有稳定 ID、别名和所属主题。

### ASA-0104 词包人工审核流程

- 优先级：`P1`
- 主责：ChatGPT
- 协作：用户
- 输出：新增、降级、排除、过期和版本变更规则。
- 验收：词包修改不覆盖历史版本，且记录修改原因。

## 4. Phase 1：异常检测

### ASA-0201 定义异常事件契约

- 优先级：`P1`
- 主责：Codex
- 输出：`AnomalyEvent` Schema。
- 字段：来源、指标、窗口、基线、异常类型、异常强度、质量状态和触发规则。
- 验收：异常可追溯到原始时间序列。

### ASA-0202 透明基线检测器

- 优先级：`P1`
- 主责：Codex
- 输出：滚动中位数、MAD、分位数、变化率和持续性检测。
- 验收：包含单日尖峰、持续抬升、周期性峰值测试。

### ASA-0203 平台独立阈值

- 优先级：`P1`
- 主责：Codex
- 输出：平台和指标级阈值配置。
- 验收：不得跨平台直接比较绝对指数值。

### ASA-0204 数据伪异常检测

- 优先级：`P1`
- 主责：WorkBuddy
- 协作：Codex
- 输出：平台改版、数据修订、采集失败、截图识别错误和权限变化的异常标记。
- 验收：数据问题优先输出 `DATA_ARTIFACT` 候选。

## 5. Phase 1：候选事件采集

### ASA-0301 定义候选事件契约

- 优先级：`P1`
- 主责：Codex
- 输出：`CandidateCause` 和 `EvidenceItem` Schema。
- 验收：候选原因可关联多个支持证据和反证。

### ASA-0302 官方来源采集清单

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：公司官网、官方公众号、官方微博、产品页、发布会、交易所公告的真实可用入口。
- 验收：记录账号权限、历史范围和采集方式。

### ASA-0303 公共内容源采集清单

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：新闻、微信公众号样本、微博、抖音、小红书、B站、知乎、论文、GitHub 和模型平台。
- 验收：区分官方、转载、搬运和用户讨论。

### ASA-0304 异常窗口规则

- 优先级：`P1`
- 主责：Codex
- 协作：ChatGPT
- 输出：不同平台的候选事件检索窗口。
- 验收：事件发布时间晚于异常峰值时，不得作为直接首因。

### ASA-0305 工件和来源存档

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：截图、页面快照、导出文件、哈希和采集时间。
- 验收：任意证据可以复核。

## 6. Phase 2：实体消歧与内容聚类

### ASA-0401 规则消歧基线

- 优先级：`P1`
- 主责：Codex
- 输出：基于锚点词、上下文词、排除词和实体共现的规则分类器。
- 验收：提供 `Matrix` 电影、数学、世界模型和未知内容测试集。

### ASA-0402 内容去重

- 优先级：`P1`
- 主责：Codex
- 输出：精确哈希、近重复标题和正文相似度去重。
- 验收：同一稿件被多账号搬运时，不得被计作多个独立原因。

### ASA-0403 内容聚类

- 优先级：`P2`
- 主责：Codex
- 输出：事件主题聚类和首发源候选。
- 验收：聚类必须保留原始证据，不允许只存向量结果。

### ASA-0404 大模型辅助分类

- 优先级：`P2`
- 主责：Codex
- 协作：ChatGPT
- 输出：结构化分类提示、置信度、解释和人工复核接口。
- 验收：大模型输出不得覆盖规则和人工事实；低置信度进入 `UNKNOWN`。

### ASA-0405 目标语义占比估计

- 优先级：`P2`
- 主责：Codex
- 输出：抽样或全量内容中目标语义的估计比例及置信区间。
- 验收：明确这是样本估计，不伪装成平台官方用户构成。

## 7. Phase 2：传播路径与证据评分

### ASA-0501 传播边契约

- 优先级：`P2`
- 主责：Codex
- 输出：`PropagationEdge` Schema，记录首发、转载、引用和跨平台搬运。
- 验收：区分发布时间、首次发现时间和传播峰值。

### ASA-0502 候选原因透明评分

- 优先级：`P1`
- 主责：Codex
- 输出：时间匹配、语义匹配、来源权威、平台匹配、规模、跨平台支持、传播匹配、反证和质量惩罚。
- 验收：每个总分可以拆回分项。

### ASA-0503 反证搜索

- 优先级：`P1`
- 主责：ChatGPT
- 协作：WorkBuddy
- 输出：同日更强同名事件、目标官方沉默、其他平台缺乏同步、周期效应和平台改版等反证清单。
- 验收：每份正式归因报告必须包含反证章节。

### ASA-0504 多原因归因

- 优先级：`P2`
- 主责：Codex
- 输出：多个候选原因共同解释异常的结果结构。
- 验收：保留未解释部分；分项权重不被描述为官方真实占比。

### ASA-0505 主题分数门禁

- 优先级：`P1`
- 主责：Codex
- 输出：高歧义关键词进入 ASE 综合分前的降权、隔离或拒绝规则。
- 验收：`Matrix` 原始指数不得未经消歧直接计入世界模型主题热度。

## 8. Matrix 首个案例

### ASA-0601 固化原始观察

- 优先级：`P1`
- 主责：WorkBuddy
- 输入：用户看到的微信指数日期和值。
- 输出：截图或人工证据、平台、关键词、日期、时间范围和数据质量记录。
- 验收：确认是 `Matrix`，不是 `Matrix 3.5`；确认日期为 2026-06-24 或修正后的准确日期。

### ASA-0602 补齐时间序列

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：异常日前后至少 30 日数据；如平台只显示有限区间，记录限制。
- 验收：能够判断单日峰值、趋势抬升还是周期效应。

### ASA-0603 采集锚点词对照

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：
  - `Matrix 3.5`
  - `Matrix 3.5 昆仑万维`
  - `昆仑万维`
  - `世界模型`
  - `World Model`
  - `Riemann`
  - `黎曼模型`
- 验收：明确“暂无数据”和数值零的区别。

### ASA-0604 采集同名竞争事件

- 优先级：`P1`
- 主责：ChatGPT
- 协作：WorkBuddy
- 输出：异常窗口内电影、数学、机器人、软件、游戏、加密和其他 Matrix 实体事件清单。
- 验收：每项有时间、来源和证据等级。

### ASA-0605 微信内容样本分类

- 优先级：`P1`
- 主责：WorkBuddy
- 协作：ChatGPT
- 输出：异常日前后微信内可获得的相关文章和搜索结果样本分类。
- 验收：至少区分目标世界模型、其他 AI、电影、数学、公司或机器人、未知。

### ASA-0606 跨平台对照

- 优先级：`P1`
- 主责：WorkBuddy
- 输出：百度、抖音、微博、B站、小红书、知乎同期信号。
- 验收：记录缺失，不能以缺失代替无变化。

### ASA-0607 生成案例报告

- 优先级：`P1`
- 主责：ChatGPT
- 输出：`reports/attention/attribution/cases/matrix-wechat-2026-06-24.md`
- 必含：
  - 异常概况
  - 歧义画像
  - 候选原因
  - 支持证据
  - 反证
  - 目标相关性
  - 未解释部分
  - 置信度
  - `NO_TRADE`
- 验收：证据不足时结论必须为 `INSUFFICIENT_EVIDENCE` 或 `AMBIGUOUS`。

## 9. 回测与在线化

### ASA-0701 建立历史案例集

- 优先级：`P2`
- 主责：ChatGPT
- 输出：高歧义词和低歧义词的正例、反例与数据伪异常案例。
- 验收：包含事后证明与目标主题无关的峰值。

### ASA-0702 归因准确率评估

- 优先级：`P2`
- 主责：Codex
- 输出：分类准确率、召回率、校准、未知率和人工复核一致性。
- 验收：不能只评估被系统成功归因的样本。

### ASA-0703 置信度校准

- 优先级：`P2`
- 主责：Codex
- 输出：归因置信度与真实正确率的校准曲线。
- 验收：高置信度错误必须被单独审计。

### ASA-0704 自动归因报告

- 优先级：`P3`
- 主责：Codex
- 输出：异常触发后的研究报告生成器。
- 验收：自动报告附证据、反证、质量状态和 `NO_TRADE`。

### ASA-0705 第二大脑只读卡片

- 优先级：`P3`
- 主责：WorkBuddy
- 输出：异常、候选原因、置信度、证据和后续验证项的只读卡片。
- 验收：不能反向修改原始事实和历史归因。

## 10. 第一个 Sprint

### Sprint 目标

完成 `Matrix` 的可审计人工归因闭环，不追求全自动化。

### Sprint 范围

1. `ASA-0001` 至 `ASA-0004`
2. `ASA-0101` 至 `ASA-0103`
3. `ASA-0201` 至 `ASA-0203`
4. `ASA-0301`、`ASA-0304`、`ASA-0305`
5. `ASA-0601` 至 `ASA-0607`

### 完成定义

- `Matrix` 已登记为高歧义词。
- `Matrix 3.5 昆仑万维` 已登记为低歧义锚点词。
- 原始微信指数观察有截图或人工证据。
- 至少获得异常前后 30 日中的可用时间范围。
- 候选原因和竞争语义都有证据。
- 正式报告包含反证和未知项。
- 证据不足时没有强行归因。
- 未触发任何真实交易动作。
