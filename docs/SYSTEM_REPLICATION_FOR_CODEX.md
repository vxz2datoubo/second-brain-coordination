# 第二部分: A股独有的核心认知

## 2.1 T+1的双重身份

```
规则: 今天买入→明天才能卖出; 之前买入→今天随时可卖

任何参与者仓位 = Seasoned(旧仓, T+0有能力卖) + Fresh(新仓, T+1锁)

散户: Seasoned=0, Fresh=100% → 完全T+1约束
机构: Seasoned>0 → 有能力T+0, 但做不做需其他信号判断

→ "机构有底仓=T+0能力" ✅
→ "机构有底仓=一定在做T+0" ❌ (绝对化禁止)
```

## 2.2 OHLCV资金流代理 = 无效

```
OHLCV MF vs 真DDX: 相关系数 r=0.294, R²=0.086
方向一致率: 仅43% (比抛硬币差!)

→ 禁止所有技能使用CMF/OBV/任何基于OHLCV的资金流指标
→ 唯一有效数据源: WeStock fund_flow (MainNetFlow/JumboNetFlow) + TDX DDX
```

## 2.3 四渠道不对称性

```
大单(>20万)/特大单(>100万) → 纯机构 (散户结构上无法执行)
中单(5-20万)              → 最模糊 (机构拆单+散户都有)
小单(<5万)                → 混合 (散户+机构拆单伪装)

→ 大单永远真实, 小单需警惕拆单伪装
→ 共识(大小单同向) = 3.8倍增强
```

## 2.4 锁仓深度 → 自适应校准模式

```
锁仓>60% (如昆仑71%): 名义阈值模式
  → 活跃盘薄, 名义换手≈真换手, 校准边际收益小

锁仓<50% (如蓝标34%): 强制真换手校准模式
  → 活跃盘厚, 名义"缩量"可能为假象, 校准后SupplyTest翻倍(+1.86%→+3.73%)
```

## 技能 #1: a-stock-trading-hours (A股交易时段)

**职责**: 定义交易时间 + T+1基础规则 + 机构底仓vs散户区分

**输出信号**:
- 时段判断 (竞价/上午/午休/下午/尾盘)
- T+1规则判定 (该参与者是"实质性T+0"还是"真T+1")

**关键规则**:
```
集合竞价: 9:15-9:20可撤单(烟雾弹), 9:20-9:25不可撤单(真意图)
大单>500手卖方 → 绝对不可能是散户 (结构限制)
大单>500手买方 → 绝对不可能是散户
```

**联动**: 所有技能都依赖此技能的T+1认知基础

## 技能 #2: t1-lockup-tracking (T+1锁仓追踪)

**职责**: 追踪四类筹码 + 换手衰减 + 解锁抛压 + 隔夜折价 + 每日筹码画像 + **Seasoned/Fresh分离**

**输出信号**: 
- 坚定筹码比率 (0-100%)
- 明天预期抛压 (万手)
- Seasoned/Fresh分离度
- 机构T+0最大容量 (T0_RATIO)

**五引擎 + Seasoned模型**:
```
引擎1 — 四类资金累积追踪 (特大/大/中/小单净买累积)
引擎2 — 换手率衰减 (筹码在集中/发散)
引擎3 — 不坚定筹码抛压量化 (盈亏区间加权 → 来自anchoring)
引擎4 — 隔夜折价检测 (T+1制度性折价 vs 真恐慌)
引擎5 — 每日筹码画像 (坚定度评分 0-100)

Seasoned/Fresh模型 (§四B):
  CUM_DDX(Seasoned) = 前N日累积DDX净买
  TODAY_DDX(Fresh) = 今日DDX
  T0_RATIO = Seasoned/(Seasoned+Fresh) → 最大T+0能力(不=实际行为)
  
  昆仑7/9: T0_RATIO=85.9% (有能力, 但实际行为需DDY判断)
```

**联动**: anchoring(盈亏区间) / lockup-depth(锁仓深度) / turnover-cap(真换手)

## 技能 #3: turnover-cap-analysis (换手率硬上限)

**职责**: T+1下每票每日最多交易1次 → 换手率直接暴露锁仓深度

**核心公式**:
```
活跃浮盘 = 流通股 × (1 - 锁仓深度) → 来自lockup-depth
真换手 = 名义换手率 ÷ 活跃盘占比

昆仑7/9: 名义14.3% ÷ 活跃29% = 真换手55%
→ 超过一半的活跃浮盘今天交易了
```

**输出信号**: 锁仓深度(0-100%) / 真换手率 / 真实活跃盘量

**自适应规则**:
```
锁仓>60%: 直接使用名义换手阈值
锁仓<50%: 强制使用真换手校准
蓝标实证: 校准后SupplyTest 5日收益 +1.86%→+3.73%(翻倍)
```

## 技能 #4: lockup-depth-analysis (筹码锁仓深度)

**职责**: 四层锁仓模型 → 活跃盘 + 控盘率 + 脆弱盘

**四层模型**:

        主力对活跃盘控盘率: 15%/29%=52% 🔥

**核心洞察**: 42%锁仓=85%控盘 (主力不需控股, 只控可交易筹码)

**输出信号**: 锁仓深度 / 活跃盘比例 / 控盘率 / 脆弱盘占比

## 技能 #5: supply-test (供应测试)

**职责**: 检测缩量回落=供应枯竭, 区分真缩量vs假缩量

**核心判断**:
```
缩量回落 + DDX正 → 推测机构可能在买Fresh + 可能没动Seasoned → 倾向看多
缩量回落 + DDX平 → 机构可能不在场 → 信号减弱
缩量回落 + DDX负 → 机构可能在卖Seasoned → 危险

锁仓校准(来自lockup-depth):
  >60%锁仓: 名义缩量信号可信度90%
  <50%锁仓: 需真换手校准
```

**T+1适配**: 机构从Seasoned可卖, 缩量=推测主动不动用(不是被动被锁)

**输出信号**:
```
+1: 供应测试通过 (缩量+DDX正+锁仓>60%)
 0: 不确定 (缩量但DDX平或锁仓中等)
-1: 供应测试失败 (放量下跌或DDX负)
置信度: 锁仓>60%→90%; 锁仓<50%→60%
```

# 第四部分: 概率融合层 (PROBABILITY FUSION)

## 4.1 为什么需要这一层

25个技能各自输出方向(+1/0/-1), 但没有任何一个输出概率。融合层解决"可能是/倾向于"的模糊性。

## 4.2 核心公式 (基于ETIA 2026)

```
Fusion_Score = Σ(w_i × S_i)  for i = 1..n

  w_i = 技能权重 (基于249天回测)
  S_i = 技能信号 (+1, 0, -1)

P(bullish) = Sigmoid(Fusion_Score × scale_factor)
  → 映射到五级分类
```

## 4.3 权重表 (昆仑249天回测)
