# Memory Authority Boundary

## Purpose

`QUANT-SYSTEM-MASTER-MEMORY.md` and related long-term memory documents are navigation aids, indexes, and conflict prompts.

They are not runtime configuration.

## Authority Order

When conflicts appear, apply this order:

1. current explicit user instruction
2. `F:/aidanao/AGENTS.md` and approved subdirectory rules
3. approved schema and configuration
4. code and tests
5. permanent memory documents as navigation and conflict hints

## Runtime Source Of Truth

The active mother-system runtime remains in:

- `F:/aidanao/brain_core`
- `F:/aidanao/bulletin`
- `F:/aidanao/data`

Permanent memory must not silently override:

- formal schemas
- approved code paths
- trading rule engines
- validated configuration

## Bridge Rule

Future memory changes should arrive as an explicit memory patch or governance proposal.

They should not directly rewrite active runtime storage or schema definitions.
