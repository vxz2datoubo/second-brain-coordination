# 历史教训库 | 2026-07-05

## 教训: 追高买入 (2026-07-02)
现象: 昆仑高开后追买，当日反转被套
教训: 开盘30分钟内禁止买入，等方向确认
教训: 大盘下跌趋势禁止追高

## 教训: 卖出后立即接回 (2026-07-03)
现象: 46.45卖出昆仑，45.81接回，盈利变亏损
教训: 卖出后至少等30分钟再判断是否接回
教训: 卖出后等待是纪律，不是错过机会

## 教训: 涨停次日追高 (历史)
教训: 涨停次日不追高，溢价风险大

## 系统经验: Microsoft Store / MSIX 安装故障恢复 (2026-07-17)
现象: 官方 ChatGPT 新版整合包可以被 msstore 源发现，但安装阶段反复报 0x80004002；旧 Codex 仍需保留，不能用卸载旧版来换取新版本。
第一性原理: 安装不是一个“地区开关”，而是一条链：地区可见性 -> 账户授权 -> 包身份 -> 内容下载 -> AppX 暂存 -> 注册 -> 启动。每一层都要单独验证。
关键教训: Windows HomeLocation=美国只能证明本机地区元数据，不等于 Microsoft 账户市场、缓存许可证市场或 Store 下载链路已经正常。
关键教训: 当 AppX 日志出现 Msixvc、DownloadingProduct、No such interface supported，必须检查 InstallService、Delivery Optimization(DoSvc)、AppXSvc、BITS 和系统组件，而不是反复修改语言/地区。
恢复原则: 先收集事件日志，再做可回滚修复；缓存目录优先改名备份，不直接删除；DISM/SFC 报待重启时必须先重启再继续安装。
保留原则: 安装新包前记录旧 AppID/包身份；新包验证成功后，必须再次确认旧入口仍存在。
边界: 不把一次本机修复经验当成所有地区、账户或 Store 后端都必然适用；不安装非官方重打包程序，不打印凭据。
