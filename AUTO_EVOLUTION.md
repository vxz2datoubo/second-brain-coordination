## 四、月级进化

### 4.1 每月末任务

| 任务 | 说明 | 产出 |
|------|------|------|
| 胜率验证 | 自检验引擎验证系统胜率 | 验证报告 |
| 参数更新 | 根据月数据调整参数 | 新配置 |
| 策略迭代 | 评估28个策略模块表现 | 策略更新 |
| 年度回顾 | 汇总月度数据 | 年报 |

### 4.2 自检验机制

#### 核心验证项

1. **一致性检查**
   - 新结论与已有记忆是否矛盾
   - 规则更新是否与历史教训冲突

2. **溯源验证**
   - 检索到的知识能否追溯到原始来源
   - 来源可信度评分

3. **置信度评分**
   - 每条记忆基于来源质量评分
   - 交叉验证新鲜度

4. **推理链验证**
   - 多步推理是否自洽
   - 决策路径是否可追溯

5. **A/B决策审计**
   - 决策路径跟踪
   - 结果对比

### 4.3 迭代式真相发现

基于 Li et al. (VLDB 2014) 框架：

```
每条记忆可信度 = f(来源权重, 交叉印证, 图谱连接, 内容质量)
来源权重 = -log(error + ε)  ← 靠谱来源→高权重

迭代至收敛 (Δw < ε)
```

## 五、进化触发条件

### 5.1 自动触发

| 条件 | 触发动作 | 优先级 |
|------|----------|--------|
| 日亏>2% | 立即熔断，全天停止 | P0 |
| 连亏3笔 | 全天停止交易 | P0 |
| Regime=EXTREME | 立即平所有T仓 | P0 |
| 某因子胜率<40% | 自动降低该因子权重10% | P1 |
| 某因子胜率>70% | 自动提高该因子权重5% | P1 |
| 连续2日亏损 | 降仓至1槽 | P1 |
| 连续3日盈利 | 考虑加仓10% | P2 |

### 5.2 手动触发

| 命令 | 说明 |
|------|------|
| `POST /api/evolve/evaluate` | 执行一次全面评估 |
| `POST /api/evolve/apply-rules` | 应用优化规则 |
| `POST /api/evolve/auto-fix` | 自动修复问题 |

## 七、决策前自检

### 7.1 决策咨询流程

```
用户决策 → DecisionConsult.consult()
              │
              ├─→ 扩展查询关键词
              │
              ├─→ 搜索教训库 (category=decision-lessons)
              │
              ├─→ 搜索规则库 (category=rules)
              │
              ├─→ 搜索知识库
              │
              └─→ 判断是否有红线类教训
                      │
                      ├── 有红线 → 输出警告
                      └── 无红线 → 输出建议
```

### 7.2 使用示例

```python
from core.decision_consult import DecisionConsult

dc = DecisionConsult()
result = dc.consult("我想在开盘追高买入昆仑", context={"大盘": "下跌"})

if result.has_warnings:
    print(result.format_warnings())
    # 输出:
    # 🔍 决策前检索: 我想在开盘追高买入昆仑
    # 
    # ⚠️ 匹配到 2 条历史教训:
    #   • [教训] 追高买入
    #     → 开盘30分钟内禁止买入，等方向确认
    #   • [教训] 大盘下跌趋势禁止追高
    # 
    # 🚨 警告: 这个动作有反复犯的教训!
```

## 八、进化引擎评分

### 8.1 综合评分 (0-100)

| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 知识丰富度 | +15 | 节点数/10 + 边数/5 |
| 用户满意度 | +20 | 好评率 × 20 |
| 活跃度 | +10 | 交互数/5 |
| 错误惩罚 | -20 | 错误数 × 3 |

### 8.2 洞察生成规则

- 知识库节点<5: 建议多导入知识
- 无关联边: 建议建立关联
- 关联密度<30%: 检索覆盖度不足
- 好评率≥80%: 系统表现优秀
- 好评率≤40%: 需重点关注

## 十、最佳实践

1. **每日必须**: 收盘后记录日志，分析教训
2. **每周必须**: 周五执行周级进化，更新规则
3. **每月必须**: 月末执行自检验，验证胜率
4. **决策前必检**: 任何交易决策前调用DecisionConsult
5. **异常必记**: 任何亏损都必须记录教训
6. **成功必享**: 成功经验也要记录供后续参考
