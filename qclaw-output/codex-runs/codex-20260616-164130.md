设计思路：`decision_consult.py` 把“做决策前先查第二大脑”封装成一个轻量客户端：输入决策主题后扩展多组检索词，分别拉取历史教训、规则和相关知识，按 `id` 去重、按 `score` 排序，再用高分或红线关键词生成警告，最后通过 `ConsultResult` 统一返回并支持格式化输出。

做得好的地方：
- 结果模型独立：`ConsultResult` 把匹配项、警告状态和展示格式集中管理，调用方使用简单。
- 检索流程分层清楚：`_expand_queries`、`_search`、`_is_real_lesson`、`consult` 职责基本分开。
- 有风险前置意识：优先检索 `decision-lessons`，并对高分/红线类教训触发 warning，符合“决策前拦截”的目标。

可改进点：
- 文件当前显示存在明显编码错显，且部分行看起来被破坏，例如 [decision_consult.py](F:/ai/core/decision_consult.py:33)、[decision_consult.py](F:/ai/core/decision_consult.py:122)、[decision_consult.py](F:/ai/core/decision_consult.py:139)，建议先确认源码编码和语法完整性。
- `_search` 捕获所有异常后直接返回空列表，会吞掉 API 不可用、JSON 错误、超时等问题，调试和告警会比较困难。
- 有一些硬编码和未使用内容，例如 `DEFAULT_LESSON_QUERY_TEMPLATES`、黑名单、风险关键词、分数阈值，后续适合抽成配置，并避免直接用 `h["id"]` 这类可能触发 `KeyError` 的访问方式。
