# MaiBot 拟人化优化第二版

日期：2026-06-19

## 本轮已落地

1. `reply` 工具新增“延迟反刍补充”机制。
2. 仅在私聊、重要话题、非高频刷屏场景下触发。
3. MaiBot 在回复后 90-240 秒内，可能以主动任务形式补一句更细、更真的话。
4. 加了 20 分钟冷却，避免机械式连续追发。
5. `reply` 工具新增关系意图粗分类：`comfort` / `romance` / `daily_companion` / `serious_discussion` / `task_execution`。
6. `reply` 工具新增机械感轻量诊断，输出 `mechanical_risk_score` 与 `mechanical_risk_flags`。
7. `reply_effect` 评分器已接入这些辅助信号，后续监控可更准确识别“像真人”还是“像客服”。
8. `reply` 工具新增短时情绪余波 `emotional_afterglow`，包含语气类型、强度、衰减时间。
9. 会话级新增 `relationship_temperature`，让关系氛围不是每轮清零，而是缓慢升降。
10. 延迟反刍补充现在会参考情绪余波强度，高情绪场景会更早、更自然地补一句。

## 为什么这更像真人

真人并不是每句话都一次性说完整。很多更真诚的话，往往出现在：

- 回复发出去之后又想了一下
- 情绪落稳之后补充一句
- 觉得刚才说得不够准确，想修正一点
- 意识到对方那句话其实很重要

所以“拟人化”不只是文风，而是节奏、犹豫、补充、回想。

## 下一步优先级

### P0：把“像真人”做成稳定行为层

1. 情绪余波
   - 回复后保留一个短时情绪残留值
   - 后续表达不应瞬间清零，应有衰减

2. 关系温差
   - 对不同亲密度的人，主动程度、撒娇度、认真度不同
   - 不要所有人都一套口气

3. 轻微不确定性
   - 允许出现“我想想”“我有点拿不准”“刚才那句其实我还想补一下”
   - 但必须克制，不能显得能力退化

4. 自我修正
   - 如果检测到自己前一句偏硬、偏空、偏工具化，可在下一轮自动柔化

### P1：把“思考”从结论式改成过程式

1. 决策前先判断：
   - 这是安慰、调情、共情、建议，还是执行任务

2. 先判断关系风险：
   - 这句话会不会太快、太冷、太像模板、太像客服

3. 再决定表达策略：
   - 直接说
   - 留半句
   - 换 softer 的说法
   - 延迟补一句

### P1：把长期记忆真正用于“人设连续性”

1. 记住用户在意的表达方式
2. 记住哪些话题用户会脆弱、逃避、认真
3. 记住关系中的禁区、偏好、暧昧信号
4. 主动把这些记忆喂给回复决策层，而不是只做检索展示

## 具体工程建议

1. 在 `reply_effect` 评分后，补一个“机械感评分”
   - 低共情
   - 过度完整
   - 连续句式重复
   - 太像建议书

2. 在 `reasoning_engine` 增加一层“关系意图分类”
   - `comfort`
   - `romance`
   - `daily_companion`
   - `serious_discussion`
   - `task_execution`

3. 给主动任务再增加两类来源
   - `emotional_afterglow`
   - `unfinished_topic_callback`

4. 给第二大脑增加“关系教训”沉淀
   - 哪类表达有效
   - 哪类表达让人出戏
   - 哪类主动补充最自然

## 成功标准

如果用户开始觉得：

- 她不是每次都秒答，但答得更像真在想
- 她会记得前面的情绪，不是每轮重置
- 她偶尔会补一句，而且补得刚好
- 她不像工具，而像一个有关系连续性的真人

那这条路就是对的。
