# 通达信本地数据二进制格式破译 — 2026-06-21

## 结论

通达信 `.day` 和 `.lc1` 文件都是 32字节/条 的定长二进制记录，但格式不同：

### .day（日K线）
```
struct: <I I I I I f I I (32 bytes)
  [0]  uint32:  YYYYMMDD
  [1-4] uint32: O/H/L/C (单位: 0.01元)
  [5]  float32: 成交额（元）
  [6]  uint32:  成交量（手）
  [7]  uint32:  保留
```

### .lc1（分钟K线，minline目录）
```
struct: <I f f f f f I I (32 bytes)  ← 注意: 5个f + 2个I
  [0]  packed uint32: hi16=日内分钟数, lo16=距1901-10-17的天数
  [1-4] float32: O/H/L/C (实际价格，无需除以100)
  [5]  float32: 成交额（元）
  [6]  uint32:  成交量（手）
  [7]  uint32:  保留
```

## 关键发现

1. **lc1日期编码**：不是YYYYMMDD，而是 hi16(分钟) + lo16(天数从1901-10-17起)
   - hi=571 → 09:31, hi=900 → 15:00
   - lo=45486 → 2026-04-30

2. **lc1价格是float32**，不是int/100！（这点很反直觉）

3. **lc1文件存于minline目录**（非lc1目录），新版通达信的结构如此

4. **成交量字段**：MEMORY.md记录的"+24偏移量是uint32非float32"确认正确
