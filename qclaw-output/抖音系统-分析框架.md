## 分析框架补充

### 1. 评论区分析维度

- **情绪**：正向/负向/中性三分类 + 强度（0-1），用高频词+表情符号做快速聚类，识别整体舆论温度
- **矛盾**：提取争议话题（正反观点碰撞点），统计支持/反对比例，标记潜在公关风险等级
- **需求**：从评论中挖掘"要是能XX就好了"类表达，归类为功能需求/内容需求/体验需求
- **市场机会**：交叉"高需求+低供给"象限，识别竞品未覆盖的空白点，输出机会卡

### 2. 关键帧提取策略

- **场景切换检测**：基于直方图差异阈值（默认30%），画面突变处取首帧，避免同场景重复
- **信息密度优先**：含文字字幕/数据图表/人物特写的帧优先保留，空镜/转场/模糊帧自动过滤
- **黄金比例采样**：每N秒取1帧（默认5s）+ 高潮片段加密（弹幕峰值区间每1s取1帧），总量控制在20-40帧/视频

### 3. 文件命名规范

```
{YYYYMMDD}_{平台}_{作者}_{内容类型}_{序号}.{ext}
例：20260413_bilibili_某UP主_关键帧_01.png
```

- 平台代号：bilibili / douyin / xiaohongshu / youtube
- 内容类型：关键帧 / 评论截图 / 分析报告 / 原始数据
- 禁止特殊字符和空格，下划线分隔，全小写英文

### 4. 第二大脑集成

- **统一API入口**：POST `localhost:8766/api/digest/text`，payload 包含 title + content + tags，返回 digest_id
- **分类标签体系**：分析结果打标 `#内容分析 #平台名 #作者`，原始素材打标 `#raw #来源 #日期`
- **关联回链**：每条分析结果附带 `source_url` 和 `source_digest_id`，形成分析→原始素材的双向追溯链
