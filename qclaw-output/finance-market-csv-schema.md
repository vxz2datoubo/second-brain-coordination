# A股日线 CSV 格式

每个标的一个 CSV，建议文件名:

```text
代码-名称.csv
300750-宁德时代.csv
688981-中芯国际.csv
```

必需列:

```csv
date,open,high,low,close,volume
2026-06-20,10.00,10.50,9.80,10.30,1000000
```

建议增加 A 股实盘校验列:

```csv
date,open,high,low,close,volume,adjustment,limit_up,limit_down,paused
2026-06-20,10.00,10.50,9.80,10.30,1000000,qfq,11.33,9.27,0
```

字段含义:

- `adjustment`: 复权方式，建议填 `qfq`、`hfq` 或 `raw`。
- `limit_up`: 当日涨停价。
- `limit_down`: 当日跌停价。
- `paused`: 是否停牌，`1/true/停牌` 表示停牌，`0/false` 表示正常交易。

也支持中文列名:

```csv
日期,开盘,最高,最低,收盘,成交量
2026-06-20,10.00,10.50,9.80,10.30,1000000
```

质量检查会读取这些可选列。缺少复权、涨跌停价或停牌字段时不会直接判失败，但会生成警告；真实交易前必须确认数据源、复权方式、停牌和涨跌停状态。

仓位计划会使用这些字段做执行拦截:

- `paused` 为停牌时，不允许新开仓。
- 最新收盘价接近或触及 `limit_up` 时，不允许新开仓，因为可能买不进。
- 最新收盘价接近或触及 `limit_down` 时，不允许新开仓，因为流动性和止损执行风险过高。
