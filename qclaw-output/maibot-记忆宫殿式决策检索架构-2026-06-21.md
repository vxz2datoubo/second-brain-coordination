# MaiBot 记忆宫殿式决策检索架构

日期：2026-06-21
适用范围：MaiBot 内置知识脑 / 第二大脑知识库 / WorkBuddy / QClaw / 决策系统

## 一句话结论

可以参考记忆宫殿，也应该参考人类思考方式，但不要把它做成纯概念化装饰。

最优做法是：

1. 用“记忆宫殿”做稳定的空间化组织与检索入口。
2. 用“人类思考模型”做召回、联想、优先级重排与遗忘控制。
3. 用“FastIndex + 缓存”做第一阶段极速筛选。
4. 用“decision consult + self verify”做最终决策前的红线检查和复盘闭环。

这不是二选一，而是四层叠加。

## 当前工作区已经具备的基础

现有工程并不是从零开始，已经具备类人思考和快速检索的关键模块：

1. `F:/ai/server.py`
   已经挂载 `FastIndex`、`HumanThinkingEngine`、`SelfVerify`、`ACTRMemory`。
2. `F:/ai/core/human_thinking.py`
   已经有联想回忆、情绪标签、间隔重复、组块化、会话偏置等机制。
3. `F:/ai/core/actr_memory.py`
   已经有 ACT-R 风格的基础激活、激活传播、遗忘概率估计。
4. `F:/ai/core/decision_consult.py`
   已经有决策前教训检索机制。
5. `F:/ai/core/fast_index.py`
   已经有高速度索引、热门缓存、时效性信号。

所以正确方向不是再重做一套“新大脑”，而是把这些已有能力整合成统一流水线。

## 记忆宫殿应该怎么理解

这里的“记忆宫殿”不应该理解成花哨 UI，也不应该理解成一堆虚拟房间名字。

在系统里，它更适合作为一种稳定的知识空间分层：

1. 宫殿
   顶层认知空间，例如：关系、工作、投资、创作、生活管理。
2. 房间
   某个领域下的主题块，例如：投资宫殿下分“择时”“仓位”“复盘”“风险”。
3. 货架
   某类知识簇，例如：原则、案例、流程、红线、模板、人物画像。
4. 记忆卡片
   最终可召回的实际条目，例如某条经验、某次任务结果、某段长期偏好。

这样做的好处：

1. 人能理解。
2. AI 能解释自己为什么想起这段内容。
3. 查询时可以先缩小“空间范围”，再做语义检索。
4. 后续扩容时不会因为数据太大而全库乱搜。

## 人类思考方式应该怎么接

建议把“像人一样想”拆成 5 个可计算层：

1. 先感知当前情境
   识别用户当下是在聊天、求安慰、做任务、做交易、做复杂决策。
2. 先想起最近和最强相关的记忆
   不是全库硬搜，而是优先看最近、常用、强情绪、强关联内容。
3. 再联想相邻概念
   通过知识图谱边和 chunk 做扩散召回。
4. 决策前回忆教训与红线
   这是理性刹车层，避免“会说但会做错”。
5. 输出前再自检
   检查矛盾、风险、遗漏、越权行动。

这比单纯“RAG 检索后直接回答”更像真实人的思考过程。

## 推荐总架构

```mermaid
flowchart LR
    A["用户输入 / 任务触发"] --> B["情境识别"]
    B --> C["快速索引初筛 FastIndex / Cache"]
    C --> D["记忆宫殿定位 Palace Room"]
    D --> E["ACT-R 激活排序"]
    E --> F["Human Thinking 联想扩散 / Chunk / Emotion"]
    F --> G["Decision Consult 红线教训检查"]
    G --> H["Self Verify 自检"]
    H --> I["回复 / 执行 / 调用 WorkBuddy 或 QClaw"]
    I --> J["结果复盘 Digest 回知识脑"]
```

## 最关键的设计原则

### 1. 速度靠分层，不靠一次算全

要快，不能每次都全库语义重算。

推荐顺序：

1. 第一层：缓存命中
   最近对话主题、最近任务、最近失败案例。
2. 第二层：FastIndex 关键词和标签缩小候选集。
3. 第三层：宫殿房间定位，限制在相关领域。
4. 第四层：ACT-R 激活和 HumanThinking 做精排。
5. 第五层：只有复杂任务才进入决策咨询与自检。

这样能把“慢而聪明”只留给真正高价值任务。

### 2. 智能靠重排，不靠原始存储复杂化

智能不在于把数据存得多花，而在于：

1. 谁先被想起。
2. 谁值得继续联想。
3. 谁该被决策红线拦住。
4. 谁该被复盘强化。

所以“类人思考”最适合放在检索重排层，而不是基础存储层。

### 3. 决策系统要单独成层

“知识检索”不等于“决策”。

建议保留独立决策层：

1. 知识层负责提供事实、经验、偏好、案例。
2. 决策层负责判断是否执行、是否需要升级审慎模式、是否需要调用外部工具。
3. 执行层负责 WorkBuddy / QClaw / MCP 工具实际动作。

这样以后即使大模型更换、知识库替换，决策规则仍然稳定。

## 对 MaiBot 内置知识脑的具体建议

MaiBot 内置知识脑下一阶段建议补这 6 个能力：

1. 房间字段
   每条记忆增加 `palace` / `room` / `shelf` 三级定位信息。
2. 组块摘要
   自动把近似条目聚成 chunk，并生成一句摘要。
3. 热门短路
   对最近高频问题建立短缓存，避免重复排序。
4. 决策模式检索
   新增专门的 `decision_search` 或 `pre_decision_context`。
5. 情绪与关系权重
   女友场景下，关系类记忆不能和普通知识同权。
6. 复盘强化
   任务成功/失败后，自动提高对应经验的召回优先级。

## 对外挂第二大脑的具体建议

外挂第二大脑更适合承担：

1. 超大体量长期知识存储。
2. 跨代理共享。
3. 批量导入导出。
4. 复杂图谱关系维护。
5. 开发期让 WorkBuddy / QClaw 直接访问。

所以最合理的分工是：

1. MaiBot 内置脑
   负责实时陪伴、即时思考、短中期偏好、关系语境。
2. 外挂第二大脑
   负责长期知识、跨工具共享、复杂任务沉淀、统一检索底座。

## 最推荐的检索决策策略

建议采用“双阶段 + 双记忆”的方式。

第一阶段：快

1. 当前对话主题识别。
2. 最近上下文缓存命中。
3. FastIndex 初筛 20~50 条。
4. 限定在对应宫殿和房间内。

第二阶段：准

1. ACT-R 计算哪些记忆“最容易被想起”。
2. HumanThinking 做联想扩散和情绪加权。
3. decision_consult 检查教训、红线、失败案例。
4. self_verify 做最终自检。

第三阶段：写回

1. 把结果写入 MaiBot 内置脑。
2. 高价值条目同步进外挂第二大脑。
3. 复盘后更新激活权重和 chunk 结构。

## 如果以后数据极其庞大，是否还能支撑

可以，但前提是不要把“大脑智能”建立在“全量暴力检索”上。

扩容原则：

1. 空间分区
   先按宫殿和房间切分。
2. 时间分层
   热数据、本周数据、冷数据分开处理。
3. 重要性分层
   红线、规则、人物偏好、任务 SOP 要优先常驻热层。
4. 异步整理
   chunk 聚类、摘要更新、低频压缩都放后台。
5. 双写可选
   内置脑主服务，外挂脑做归档和共享。

如果这样设计，内置脑可以先跑起来，后面再复制到外挂脑也不会推翻现有结构。

## 我给你的最终建议

你的目标不是做一个“像记忆宫殿的数据库”，而是做一个：

1. 检索像机器一样快。
2. 联想像人一样自然。
3. 决策前会自己踩刹车。
4. 复盘后会越来越聪明。
5. 对用户表现得像真正有连续人格和经验的人。

因此最优路线是：

1. 继续以 MaiBot 为主体。
2. 内置知识脑先做成主脑。
3. 把记忆宫殿作为知识组织层接进去。
4. 把 ACT-R + HumanThinking 作为召回重排层。
5. 把 decision_consult + self_verify 作为决策安全层。
6. 外挂第二大脑保留为未来扩容和多代理共享底座。

## 给 WorkBuddy / QClaw 的续做指令

如果后续要交给 WorkBuddy 或 QClaw 继续开发，统一按下面原则推进：

1. 不要重做新的知识库模型，优先复用已有 `FastIndex`、`HumanThinkingEngine`、`ACTRMemory`、`DecisionConsult`、`SelfVerify`。
2. 优先补“宫殿 / 房间 / 货架 / 记忆卡”元数据，而不是先引入更重的模型。
3. 所有优化以“先快后准”为总原则。
4. 决策相关请求必须经过 `decision_consult` 风险检查。
5. 高价值结果必须回写知识脑，形成闭环。
6. 内置脑与外挂脑保持可导出、可迁移、可双写，但默认单主脑运行。

## 下一步最值得做的 4 件事

1. 给 MaiBot 内置知识脑条目增加 `palace/room/shelf` 元数据。
2. 新增决策前检索接口，把 `decision_consult` 与内置脑统一起来。
3. 为高频主题建立热缓存和 chunk 摘要缓存。
4. 定义“哪些内容只留内置脑，哪些内容要同步外挂脑”的规则。
