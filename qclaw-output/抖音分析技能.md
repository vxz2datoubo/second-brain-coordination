# 抖音视频分析技能流程设计

---

## 1) 流程步骤

```
用户发送抖音链接
    │
    ▼
[Step 1] 链接解析
  ├─ 识别链接类型（视频/图文/合集）
  ├─ 提取视频ID
  └─ 判断是否需要登录（私密/好友可见→提示用户）
    │
    ▼
[Step 2] Playwright 页面抓取
  ├─ 打开链接，等待页面加载
  ├─ 提取：标题、文案、标签、发布时间、互动数据
  ├─ 提取：前50条热评（按点赞排序）
  ├─ 截图：封面/关键帧
  └─ 输出 JSON → 传给 QClaw
    │
    ▼
[Step 3] QClaw 三线程分析
  ├─ 线程A：事实纠错
  │   └─ 对照公开知识库，标记疑似错误/误导信息
  ├─ 线程B：评论洞察
  │   └─ 评论情绪分布、用户核心关注点、争议点提取
  └─ 线程C：市场/行业洞察
      └─ 选题趋势、竞品动态、可复制方法论
    │
    ▼
[Step 4] 输出组装
  ├─ 按模板生成 Markdown
  ├─ 存入 F:\ai\qclaw-output\
  └─ 推入第二大脑 API（可选）
```

---

## 2) QClaw 分析 Prompt

```
你是抖音内容分析专家。基于以下抓取数据，完成三维分析：

【抓取数据】
- 标题：{title}
- 文案：{caption}
- 标签：{tags}
- 互动：👍{likes} 💬{comments} 🔗{shares} ⭐{collects}
- 热评TOP20：{top_comments}

═══════════════════

任务A：事实纠错
对视频中的事实性陈述逐条核查。若发现与公开可信数据矛盾的内容，标注：
- 原文引用
- 正确信息
- 可信来源
无错误则输出「未发现明显事实错误」。

任务B：评论洞察
- 情绪分布：正/负/中比例
- TOP3用户关注点（高频关键词+代表性评论）
- 争议点/未满足需求（如有）
- 创作者互动质量评价

任务C：市场洞察
- 该选题在当前赛道的竞争力
- 可复用的内容方法论（结构/节奏/钩子）
- 对同类创作者的启发建议

═══════════════════

输出要求：每项3-5条要点，简洁，数据驱动。
```

## 4) 技术实现要点

| 环节 | 工具 | 关键点 |
|------|------|--------|
| 页面抓取 | Playwright | 绕过反爬：模拟移动端UA、等待关键DOM、处理验证码/滑块 |
| 评论提取 | Playwright | 滚动加载评论、按点赞排序抓取、处理折叠评论 |
| 数据分析 | QClaw | 三线并行评估、交叉验证 |
| 知识存储 | 第二大脑 | `POST http://localhost:8766/api/digest/text` |
| 文件落盘 | 本地 | `F:\ai\qclaw-output\抖音分析_{date}_{id}.md` |

## 5) 后续扩展方向

- **批量分析**：用户提供创作者主页，批量抓取最近N条视频逐一分析
- **趋势追踪**：定时抓取特定标签/话题下的热门视频变化
- **竞品对比**：同时分析同类账号，输出横向对比报告
- **评论回复辅助**：基于评论洞察生成回复建议
