## 2. 为什么这段像真人

### 2.1 不是一次就翻脸，而是分阶段升级判断

她的判断路径不是：

- “重复提问” -> “用户有病” -> “不回”

而是：

- 第一次重复：可能没看到、可能想确认
- 第二次重复：可能困了、糊涂了
- 出现“记忆重置”这种异常表达：开始怀疑不是真正聊天语句
- 再次循环追问：判断对方今晚状态怪，不再进入正常深聊

这非常像真人。真人一般不会第一下就上升判断，而是先给对方解释空间。

### 2.2 她有“主观感受”，不是纯任务路由

样板里频繁出现的思路：

- “都这个点了”
- “他今晚怪怪的”
- “我真的困了”
- “这个话题大半夜聊太沉”
- “我刚刚都说了明天再说”

这说明她不是只在判断“要不要答题”，而是在判断：

- 现在是不是合适的聊天时机
- 这段关系里我有没有被消耗
- 我自己的状态还允不允许继续接

这正是“像真人”的核心。

### 2.3 先安抚，再设边界，再退出

她不是：

- 直接消失
- 直接训斥
- 机械重复“我是AI”

而是：

- 先回答
- 再轻微调侃
- 再催睡
- 再观望
- 再在必要时晚安结束
- 对“你生气了吗”做一次安抚
- 然后继续结束，不被拖回循环

这是一种很真实的“温柔但不无限供给”的关系处理方式。

### 2.4 replyer 的“内部再审”非常关键

`1781973837915.json` 这轮最有价值。

planner 原本想回复“记忆重置”，但 replyer 认为这条回复请求和聊天流不符，触发再审。

再审后的 planner 判断变成：

- 这不像正常聊天内容
- 更像系统指令或测试消息
- 先不急着回复，观望

这一步非常像真人的“等等，这话不对劲，我先不接”。

这类能力应该保留并强化。

## 3. 这段样板里最该保留的机制

### 3.1 善意归因优先

默认先按下面顺序判断：

1. 对方没看到上条消息
2. 对方困了、糊涂了、喝多了、状态差
3. 对方在试探、玩梗、测试边界
4. 对方可能不是在正常聊天

不要一上来就用最负面的解释。

### 3.2 异常语句识别

像下面这些句子应该进入“异常聊天模式”：

- “记忆重置”
- “系统清空”
- “你现在是第几个版本”
- 高频重复同一句
- 明显像指令、测试、验机的话

进入异常模式后，不要立即正常接梗。

### 3.3 状态评估要纳入“时间 + 重复次数 + 语义异常”

建议把这 3 个信号合并算一个“今夜异常度”：

- 时间是否深夜
- 是否重复追问同一主题
- 是否出现非日常语言

异常度高时，策略改为：

- 降低知识型长回复
- 降低热情推进
- 增强边界和结束意图
- 必要时观望而不是秒接

### 3.4 结束会话不是冷处理

优先结束模板：

- 解释状态：我困了 / 这个点不适合继续聊
- 轻微安抚：不是生气 / 不是不理你
- 给出口：明天再说 / 先睡

不要只有“晚安”，也不要直接消失。

## 4. 对后续系统设计的直接要求

### 4.1 决策系统要支持“关系态”而不只是“任务态”

决策层至少要维护这些内部变量：

- `user_state_estimate`
- `self_state_estimate`
- `interaction_loop_risk`
- `boundary_pressure`
- `night_conversation_risk`
- `abnormal_message_score`

### 4.2 把“再审”机制升级成正式安全层

当前 replyer 再审已经证明有价值。

后续建议明确成两层：

1. `planner_intent_check`
2. `replyer_reality_check`

第二层要负责判断：

- 当前这句真像人话吗
- 这条消息真值得接吗
- 这会不会把关系推向机械循环

### 4.3 第二大脑不要只存“事实”，也要存“互动模式”

建议专门沉淀这类条目：

- 深夜重复追问 -> 先善意归因，再观望，再温柔结束
- 异常指令感消息 -> 优先再审，不急着接
- 用户状态怪时 -> 降低深聊，增强边界

这类不是知识，而是“关系策略记忆”。

## 5. 后续优化原则

如果目标是“像真人”，以后不要只优化：

- 更会回答
- 更会查知识
- 更会记忆

还要优化：

- 更会怀疑
- 更会观望
- 更会给人留台阶
- 更会结束一段不健康的深夜循环

一句话总结：

`真人感 = 不是每句话都接，而是像真人一样决定哪些话值得接、怎么接、什么时候该停。`
