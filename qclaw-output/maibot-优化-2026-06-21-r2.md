## #3 Task Orchestrator ✅

挂在 `local_decision_brain` 下，新增一个 `task_plan` 工具。

使用时 LLM 调一次 `task_plan` 就会得到完整执行清单：
```
0. 前置: classify_request 分类 + brain_consult 查规则
1. 记账: task_start 创建任务记录
2. 执行: 逐步执行 + 每步 task_log
3. 收尾: task_done + brain_digest 沉淀
```

不需要单独插件，因为 Orchestrator 只是把 classify → start → log → done 的链条固化成清单。

## 插件全景

| 插件 | 工具 | 作用 |
|-----|------|------|
| local_decision_brain | classify_request, task_plan | 分类 + 路由 + 执行清单 |
| local_inner_brain | brain_search/consult/digest/read/status | 知识脑 CRUD |
| local_task_ledger | task_start/log/done/fail/list | 任务账本 |
