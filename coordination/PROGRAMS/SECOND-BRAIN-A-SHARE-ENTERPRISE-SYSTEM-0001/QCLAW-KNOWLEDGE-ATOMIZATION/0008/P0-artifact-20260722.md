## Key Architectural Decisions

1. **6 independent metadata fields**: privacy_class, storage_targets, transport_visibility, gpt_access, authority_level, knowledge_status. privacy_class does NOT control storage.

2. **Hard boundary only on secret VALUES**: CREDENTIAL_VALUE and FINANCIAL_VALUE atoms are permanently denied from all outputs. Metadata about secrets (where stored, field names) is knowledge allowed.

3. **Deterministic IDs from SHA-256**: Same semantic input = same ID. Enables rebuild verification.

4. **Conflicts are knowledge**: BOTH sides preserved. CONFLICT_NOT_RETURNED is a critical failure.

5. **UNKNOWN is knowledge**: Gaps documented with verification paths. Never deleted.

6. **Candidate ≠ inaccessible**: CANDIDATE_ONLY atoms have FULL_SEMANTIC_ACCESS by default.

7. **No second runtime**: QCLAW produces LearningPackets for Phase 3 runtime; never duplicates storage, retrieval, fusion, or gateway capabilities.
