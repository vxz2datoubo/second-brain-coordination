# 0017 D0 discoveries and opportunities

`agent_id: CODEX`  
`classification: findings, not implementation approval`

1. **The first real bottleneck is data admission, not a more elaborate chart pattern.** The accessible `.day` file has a stable structural hash and plausible record layout, but no declared point-in-time availability, adjustment policy, status history or publishable license. A deterministic labeler built on it now would be reproducible only on one local machine and would still be unable to justify its rule/availability assumptions.

2. **The existing blueprints already prevent the main semantic trap.** They distinguish observable breach/reclaim behavior from claims about hidden stops, participant identity and “absorption”. D1 should preserve that language; adding flow proxies would lower, not raise, evidentiary quality.

3. **The natural first vertical slice is deliberately modest.** One source-manifest-bound, synthetic PIT fixture set; prior-session zones; completed-bar breach/close-reclaim labels; explicit unknown rule/inventory paths; and no performance calculation. This gives a robust contract test before any historical performance study.

4. **No opportunity supports a stable-profit conclusion.** Even future results must compete with random levels, ordinary touches, simple breakouts, simple reversals, volatility-only and volume-only baselines, then pass point-in-time and execution constraints.
