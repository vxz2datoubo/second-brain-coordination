`agent_id: CODEX`  
`task_id: CODEX-P3-INTEGRATED-OFFLINE-MEMORY-SYSTEM-0006`

## D-001 One integrated Phase 3 package

Use one package and one Draft PR for the adapter, replay, candidate memory, retrieval and context path. This avoids a second market-data authority or parallel memory authority.

## D-002 Preserve existing authorities

- PR #51 remains the source-manifest and adapter-governance authority.
- P2 `offline_research.Bar` remains the replay bar object.
- PR #46 is the sole candidate memory implementation base.
- PR #45 contributes deterministic canonicalization, IDs, relation mapping and candidate packet tests.
- PR #52 contributes immutable, partial field evidence only.

## D-003 Local activation is narrower than source verification

An activation policy may allow read-only structural parsing only when bound to one manifest ID and one SHA256 under `LOCAL_RESEARCH_ONLY`. It cannot grant realtime, trading, raw export, authority write, or fully verified status.

## D-004 A-share temporal semantics

Daily bars use 15:00 Asia/Shanghai as `event_time`. `available_at` is generated by a versioned post-close availability policy. File mtime is audit metadata, never market time or freshness proof.

## D-005 Ambiguous vendor fields

`amount` keeps raw bytes plus float32 and uint32 candidate interpretations. `volume` keeps the vendor value while its unit remains UNKNOWN. `reserved` remains UNKNOWN. None of these fields may silently become an authoritative feature.

## D-006 Candidate memory boundary

All imported LearningPackets remain `candidate` with `authority_write=false`. Rejected, quarantined and credential values are excluded from default retrieval; conflicts and unknowns remain visible unless explicitly filtered.

## D-007 Semantic ContextBundle hash

The ContextBundle semantic hash excludes SQLite `created_at` and `updated_at` wall-clock metadata. Those fields remain available for time filtering, while the cross-run hash represents stable semantic content. This decision was added after two independent local runs exposed timestamp-only hash drift.

## D-008 Public delivery and CI boundary

GitHub CI runs only synthetic and public-safe tests on Python 3.11 and 3.13. It never accesses the local `.day` artifact. The real sample is represented only by a hash-bound aggregate receipt and candidate-only semantic objects.
