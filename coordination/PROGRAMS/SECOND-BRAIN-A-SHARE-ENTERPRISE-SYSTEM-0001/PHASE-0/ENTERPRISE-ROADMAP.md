# Enterprise Roadmap

## Phase 0 - fact freeze and plan

Deliver the present audited package. Acceptance: source-of-record map, bounded contexts, reuse decisions, dependency DAG, risks, unknowns, one selected slice and no unsafe action.

## Phase 1 - foundation contract freeze

Create a narrow compatibility layer for IDs, time, evidence, lineage, capability, quality, approval, rollback and run/trace IDs. Acceptance: fixtures, negative tests, version policy and no competing authority for the selected slice.

## Phase 2 - selected offline replay slice

Run one A-share historical/local sample through governed raw->normalised->feature->research-only signal->replay->validation->journal. Acceptance: deterministic replay, A-share constraints, costs, OOS split, no-trade output, lineage and rollback manifest.

## Phase 3 - knowledge authority slice

Implement only candidate-package validation, import quarantine, authority manifest and rebuildable read projection design after WorkBuddy #32 evidence. Acceptance: QCLAW can be offline; candidate cannot silently overwrite authority.

## Phase 4 - executable indicator/strategy slice

Promote a small, evidence-bounded indicator/combination set into versioned definitions, code, tests and OOS evaluation. Acceptance: no generic-market assumptions, parameter/timeframe evidence and abstention for unsupported fields.

## Phase 5 - temporal event intelligence slice

Audit PR #8 contract seeds, add time/leakage/claim revision gates, then run one source-authorised event pilot. Acceptance: source artifact, ambiguity handling, counter-evidence and no causal overclaim.

## Phase 6 - rule-based agent-game MVP

Start queued Issue #23 only after upstream data/replay/validation contracts exist. Acceptance: probabilistic families, dynamic advantages/weaknesses, counterfactual tests and no real-trade implication.

## Phase 7 - shadow operations

Combine bounded research outputs into a read-only shadow agenda and engineering-learning outcome loop. Acceptance: capability-aware degradation, approval gates, auditability and no direct order path.

## Phase 8+ - optional scaling

Private-Git authority, serving projection, MCP hardening, advanced simulation and any cloud deployment are separately approved projects, not automatic roadmap execution.
