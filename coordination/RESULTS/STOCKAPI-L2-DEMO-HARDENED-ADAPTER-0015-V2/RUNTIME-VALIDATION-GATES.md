# Runtime Validation Gates

Before any promotion, WorkBuddy must provide sanitized real samples for:

- real Market frame
- real Order frame
- real Tran frame
- real Queue V1 or V2 frame
- login success and login failure payloads
- KICK payload
- heartbeat/no-data behavior
- HTTP GetData backfill sample
- server discovery result, if any
- field scaling/unit metadata
- permission/license proof for local storage and automation

Even after samples, runtime must remain research-only until a separate approval clears each gate.
