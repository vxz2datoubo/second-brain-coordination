# 第二大脑与A股交易系统企业级总工程章程 v1.0

> `agent_id: GPT`
>
> 定位：整个第二大脑、A股交易研究系统、多智能体博弈、知识云、事件情报、验证风控和多AI协作的企业级母工程章程。
>
> 边界：默认 `research_only / NO_TRADE`。真实资金、券商账户、订单、凭证、生产风控和不可逆数据操作始终需要用户单独批准。

## 一、第一性原理目标

本工程不是堆积更多AI、指标和文档，而是建立一个可以持续感知、解释、推演、验证、学习和回溯的决策系统。

系统必须把以下对象连接成闭环：

```text
真实数据与事件
→ 时间、来源、质量和证据
→ 市场状态、知识和假设
→ 策略、参与者与对手模型
→ 情景、反事实和概率判断
→ 回放、成本、风控和样本外验证
→ 第二大脑记忆、冲突、失败经验和持续学习
→ 人类审批后的研究决策支持
```

系统的价值不来自“AI说得像”，而来自：

1. 每个判断可以追溯到数据、证据、模型、版本和时间；
2. 每个策略可以在A股真实制度与成本下回放；
3. 每个参与者意图只是概率假设，并保留替代解释；
4. 每次成功和失败都进入工程学习回路；
5. 第二大脑与交易系统既可独立运行，也可通过明确接口协作；
6. 真实交易执行与研究推演物理、权限和运行时隔离。

## 二、企业级系统边界

### A. 第二大脑平台

负责：

- 知识摄入、原子化、证据、版本和冲突；
- 长期记忆、项目记忆、失败经验和工程学习；
- GraphRAG、全文、结构化、向量和关系检索；
- AI上下文包、来源和置信度；
- 私有Git权威知识库与Supabase可重建服务投影；
- 旧后端与历史知识的受控适配和迁移。

### B. A股交易研究平台

负责：

- 实时行情、历史数据、Level-2聚合、公告、新闻、事件和舆情；
- 数据质量、市场阶段、时间语义和可用时点；
- 特征、指标、理论、策略、组合、实验和验证；
- 集合竞价、连续竞价、涨跌停、T+1、停牌和跨日库存；
- 回测、回放、成本、滑点、容量和风险；
- 四类参与者多智能体博弈、对手建模和反事实推演；
- 影子模式和研究决策支持。

### C. 协作控制面

负责：

- GPT、Codex、WorkBuddy、QCLAW和用户的职责边界；
- GitHub Issue、PR、活动任务索引、反馈v2和工程学习；
- 任务依赖、风险审批、版本、回滚、验收和审计；
- 不承载秘密、真实账户数据或受保护蓝图正文。

## 三、能力域与有界上下文

1. `KnowledgeAuthority`：知识权威、版本、冲突、证据和检索；
2. `MarketData`：实时、历史、供应商语义、质量和时间；
3. `EventIntelligence`：公告、新闻、政策、会议、日历和注意力；
4. `FeatureAndIndicator`：指标、理论、参数、周期和特征；
5. `StrategyResearch`：策略定义、实验、组合和适用条件；
6. `MarketSimulation`：A股撮合、制度、订单、库存和回放；
7. `AgentGame`：量化、短线活跃资金、大资金机构和散户；
8. `ValidationAndRisk`：样本外、walk-forward、成本、容量、概率校准和门禁；
9. `DecisionSupport`：情景、反情景、证据包和人工决策；
10. `EngineeringLearning`：任务预判、结果校准、正反馈强化和负反馈纠偏；
11. `AgentOperations`：多AI任务路由、权限、反馈、部署和运行监控。

每个有界上下文必须拥有：

- 明确责任；
- 系统记录源；
- 输入输出合同；
- 数据和模型版本；
- 失败、降级和回滚路径；
- 权限与安全边界；
- 观测、测试和验收标准。

## 四、系统记录源原则

不得让多个系统同时静默成为同一对象的权威。

建议目标：

- 私有Git：知识、蓝图、规则、Schema和可审查研究结论的权威历史；
- Supabase：从权威Git和批准数据投影出来的检索服务层；
- 原始行情与事件存储：不可变或追加式原始事实；
- 实验登记库：策略版本、数据切片、参数、代码、结果和验证状态；
- 工程学习登记表：预测、实际结果、意外收益、意外损害和规则变化；
- GitHub协调仓库：公开安全的任务、脱敏状态、合同、索引和验收记录。

旧系统、缓存、派生表和适配器不得被误认为新权威。

## 五、一级工作流

### W1 企业架构与治理

冻结能力地图、系统记录源、数据流、接口、权限、版本、风险和验收。

### W2 实时数据与历史回放最小闭环

优先形成：

```text
采集 → 原始记录 → 标准化 → 质量事件 → 特征 → 信号 → 回放 → 验证报告
```

### W3 第二大脑知识权威与旧后端整合

完成旧8766/8767/MCP/历史目录的受控复用、迁移和服务层设计。

### W4 指标、理论与策略知识工程

把蓝图、本体、参数和组合压成可解析、可测试、可回测的工程契约。

### W5 事件情报与注意力引擎

形成周度事件雷达、公告与新闻时间线、主题消歧和注意力因子。

### W6 四群体多智能体战略博弈

实现量化、短线活跃资金、大资金机构、散户的动态优势、弱点、合理行动、最佳反应、隐藏类型和反事实推演。

### W7 验证、风险与研究升级门

任何信号、策略或模型只有通过真实制度、成本、样本外、漂移和概率校准后，才允许升级研究成熟度。

### W8 多AI协作、部署与可观测性

明确Codex负责技术架构和核心实现，WorkBuddy负责真实环境、数据接线、部署和运行，GPT负责编排和验收，QCLAW负责知识摄入，用户负责重大方向和风险授权。

### W9 影子模式与系统学习

实时生成研究情景和验证任务，不下单。记录预测、失败、意外收益、意外损害和规则演化。

## 六、阶段路线

### Phase 0 事实冻结与企业项目计划

只做真实仓库、目录、系统、数据、代码、服务、蓝图、Issue和PR审计，产出自包含企业项目计划。

### Phase 1 企业基础合同

冻结ID、事件、证据、时间、版本、能力、权限、错误、质量和系统记录源合同。

### Phase 2 数据与回放纵向切片

打通一个标的或小样本的实时采集、历史回放、特征、信号和验证。

### Phase 3 第二大脑知识服务纵向切片

打通候选知识包、私有Git权威、可重建投影和受控检索。

### Phase 4 指标与策略工程纵向切片

选择少量真实指标和组合完成从本体到代码、测试、回测和样本外验证。

### Phase 5 事件与注意力纵向切片

选择一个主题完成事件时间线、来源、消歧、注意力和市场反应验证。

### Phase 6 多智能体规则型MVP

先实现可解释规则Agent、A股离散事件模拟和隐藏类型推断，不直接训练大规模MARL。

### Phase 7 对手建模、反事实和自博弈实验

在规则MVP通过后加入Level-k、种群、自博弈和MARL研究。

### Phase 8 统一验证与影子运行

将数据、指标、事件、策略和多智能体结果纳入统一影子模式和验证门。

### Phase 9 受控部署与持续演化

通过权限、可观测、回滚和用户审批逐步部署研究服务，真实下单仍独立审批。

## 七、架构硬规则

1. 先复用和适配，再新造组件；
2. 先规则型MVP，再复杂模型和MARL；
3. 先纵向闭环，再横向扩张对象数量；
4. LLM不直接生成订单或控制真实交易；
5. 事实、推断、假设和建议必须分层；
6. 所有时间敏感数据必须有事件时间、观察时间、进入时间和可用时间；
7. 所有重要输出必须有支持证据、反证、置信度和失效条件；
8. 不可得数据不得伪造，不可验证能力不得标记完成；
9. 任何权威迁移、权限扩大、真实资金和不可逆操作都需要用户批准；
10. 每个非简单任务必须进入工程学习与结果校准闭环。

## 八、企业级非功能要求

- 可追溯：对象、版本、输入、输出、测试和决策可回溯；
- 可复现：同一数据、代码、参数和环境可复现结果；
- 可回滚：代码、知识、配置、投影和路由有回滚方案；
- 可观测：采集、任务、模型、数据质量、失败和成本可见；
- 安全：最小权限、秘密隔离、公开与私有边界、NO_TRADE；
- 弹性：服务故障可降级到受控只读路径；
- 可扩展：模块通过合同集成，不通过共享隐式状态；
- 可校准：概率、置信度和工程预测有长期校准记录；
- 可治理：任何成熟度升级和生产化都有证据门。

## 九、成功标准

企业系统不能以“蓝图很多”作为完成标准。至少需要：

1. 企业能力图和系统记录源清晰；
2. 关键对象和接口有机器可读合同；
3. 数据、知识、策略和Agent有至少一个可运行纵向切片；
4. A股制度和真实交易成本进入仿真与验证；
5. 第二大脑能读取交易研究结果并保存证据与失败经验；
6. 四群体引擎输出概率情景而非确定性操盘故事；
7. 结果通过样本外、walk-forward和概率校准；
8. 任务、部署和模型具备可观测、回滚和工程学习；
9. 真实交易路径与研究系统保持隔离；
10. 用户可以只与GPT交互，由GitHub协调Codex和WorkBuddy执行。

## 十、当前启动策略

第一轮交给Codex的任务不是一次性实现全部系统，而是：

- 审计真实状态；
- 合并现有蓝图、Issue、PR和本地系统事实；
- 识别复用、冲突、缺口和未知项；
- 形成企业级目标架构、依赖DAG、阶段计划和首批纵向切片；
- 给出下一轮可直接拆分给Codex和WorkBuddy的任务包；
- 不在第一轮启动服务、迁移私有数据、训练大模型或触碰真实交易。
