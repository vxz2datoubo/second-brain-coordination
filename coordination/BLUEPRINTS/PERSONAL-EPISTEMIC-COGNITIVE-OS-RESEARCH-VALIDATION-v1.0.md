# PEOS研究证据与架构验证矩阵 v1.0

> `module_id: PERSONAL-EPISTEMIC-COGNITIVE-OPERATING-SYSTEM-0010`
>
> `implementation_issue: #61`
>
> 用途：把学术概念、真实系统和工程选择逐项映射，区分“已得到较强支持”“可借鉴但需本项目验证”“远期研究假设”，防止用论文名词替代工程证据。

## 一、验证等级

- `A_STRONG`: 多项经典或近期研究支持，且与本模块设计直接相关；
- `B_TRANSFER_REQUIRED`: 原领域证据较好，但迁移到个人第二大脑仍需实测；
- `C_ENGINEERING_HYPOTHESIS`: 有真实项目或新研究参考，但尚无充分证据证明适合本系统；
- `D_LONG_TERM_RESEARCH`: 当前不能承诺落地效果，只能建立合同、夹具和停止条件。

## 三、真实项目与技能联动

### 3.1 与A股机构研究和量化流程

映射为：

- 事前研究假设冻结；
- 数据可用时点和防未来函数；
- 策略/事件/主力意图只作为概率假设；
- 事后performance attribution区分市场Beta、因子、事件、执行和随机性；
- 样本外、walk-forward、成本、容量和漂移；
- 概率预测同时看校准和经济效用；
- 研究失败进入KnowledgeAtom和DecisionEpisode，而非只记录盈亏。

这使PEOS与现有交易系统W7验证、W9影子模式和工程学习形成闭环。

### 3.2 与红队和对抗评测

Issue #59负责知识原子级对抗，Issue #60负责长期检索级对抗，Issue #61负责认知画像和决策过程级对抗。

三层不能合并成一个总分：

1. 原子是否忠实；
2. 检索是否召回正确、当前、冲突和UNKNOWN；
3. 决策引擎是否抗谄媚、抗结果偏差和抗错误画像。

### 3.3 与工程学习系统

每次重要任务在执行前记录：

- 预计收益；
- 风险和负面影响；
- 关键假设；
- 成功/失败标准；
- 置信度。

执行后分别校准：

- 工程预测；
- 认知判断；
- 用户能力推断；
- 系统自身置信度。

### 3.4 与记忆宫殿

记忆宫殿只做导航索引：

- 世界模型房间；
- 用户认知特征房间；
- 决策事件房间；
- 反证与UNKNOWN房间；
- 能力训练与迁移房间。

空间邻近不得被解释为语义支持或因果关系。

## 四、必须诚实保留的UNKNOWN

1. 从微信和AI对话推断稳定认知特征的最小样本量尚无通用答案；
2. 开放学习者模型在高风险金融决策中的外部有效性需本项目验证；
3. 何种个人认知特征真正能提升决策，而非只增加解释感，尚需A/B和纵向验证；
4. LLM能否可靠识别玩笑、角色扮演、情绪宣泄和真实偏好，必须用用户数据建立专门夹具；
5. 世界模型的因果结构在政治、金融和社会系统中经常不可识别；
6. 数字分身准确度、长期漂移和自我实现预言风险尚不可承诺；
7. 认知干预可能产生反作用或用户依赖，必须观察净收益和自主性；
8. 多模型/多Agent独立性可能只是表面独立，需要防共享提示和共享证据导致相关错误。

## 五、验收方法

每项设计选择只有满足以下条件才可从蓝图升级为validated capability：

1. 有明确来源与适用边界；
2. 有机器可读合同；
3. 有正向、反向和边界测试；
4. 有真实或高保真案例；
5. 有基线对照；
6. 有失败样本和UNKNOWN；
7. 有用户纠正与回滚；
8. 有执行成本和负面影响记录；
9. 结果可复现；
10. GPT验收，不以文档数量或自报完成为准。
