# 用户指令：完整知识库与GPT全量访问

状态：ACTIVE

## 核心授权

用户授权QQ/QCLAW与第二大脑系统完整处理用户提供的全部知识，并授权GPT完整读取原子化、结构化和关联后的知识成果。

## 执行原则

1. 不得因为材料属于个人、内部、非公开、付费或许可范围而删减其语义。
2. 不得把完整内容自动压缩成只有摘要可用。
3. 不得要求已经获得用户授权的知识逐条重新审批。
4. `gpt_access`默认采用`FULL_SEMANTIC_ACCESS`。
5. `candidate / approved / superseded / conflict / unknown`描述知识真实性与演化状态，不描述GPT能否读取。
6. 存储位置与语义访问必须分离：本地或私有知识仍应通过授权知识网关完整提供给GPT。
7. 当前公开协调仓库只承载代码、合同、Schema、测试、任务状态和合成夹具；完整非公开知识正文进入本地或私有知识库。
8. 检索预算只决定一次返回多少相关知识，不得造成静默删减；遗漏内容必须明确报告。
9. 来源、冲突、修正、历史版本、例外和UNKNOWN必须保留。

## 凭证类秘密的唯一例外

以下内容不得提交到GitHub、公共对象存储、云端知识索引或普通LearningPacket正文：访问令牌、接口密钥、登录密码、会话凭证、私钥、账户认证资料及同类认证秘密。

这些内容应当：

- 只保存在用户本地受控存储中；
- 保持本地可检索和可定位；
- 在知识图谱和LearningPacket中仅保存引用ID、用途、所属系统、更新时间、校验摘要与本地位置说明；
- 不因本地保管而影响其他知识的完整学习、融合和检索；
- 仅在本地执行路径确有需要时读取，不进入公开仓库或普通上下文包。

普通API文档、接口说明、字段定义、调用方法和非认证技术资料不属于该例外，应正常完整入库。
